## 计算 n 的阶乘

### 解题思路
- **阶乘定义**：n! = 1 × 2 × 3 × … × n。
- **编程思维**：先思考解决问题的方法，再写代码。关键在于“用计算机的方法解决问题”。
- **核心逻辑**：
  - 使用循环生成 1 到 n 的数字。
  - 用一个变量（如 `ret`）累积相乘，初始值必须为 1（若为 0，结果恒为 0）。

### 代码实现要点
- 输入 n：使用 `scanf` 获取用户输入。
- 循环结构：可用 `for`、`while` 或 `do-while`，例如：
  ```c
  int ret = 1;
  for (int i = 1; i <= n; i++) {
      ret = ret * i;
  }
  ```
- **注意事项**：
  - 数据类型限制：C 语言内置类型（如 `int`、`long long`）无法处理非常大的阶乘（如 50!），会溢出。
  - **大数运算**：若需计算超大阶乘，需自行模拟（如用数组存储每一位），属于“大数运算”范畴（非海量数据，而是数值极大）。

## 计算 1! + 2! + ... + 10! 的和

### 初步实现（两层循环）
- **方法**：外层循环遍历 1 到 10，内层循环计算每个数的阶乘。
- **问题发现**：
  - 若 `ret` 变量未在每次计算新阶乘前重置为 1，会导致错误（如 3! 计算时 `ret` 残留 2! 的结果 2，导致 2×1×2×3=12 而非 6）。
  - **调试技巧**：使用 IDE 的“监视”窗口跟踪变量（如 `n`, `i`, `ret`, `sum`）的变化。
- **修复**：在内层循环前将 `ret = 1`。

### 优化实现（单层循环）
- **核心思想**：利用阶乘的递推关系——`n! = (n-1)! × n`，避免重复计算。
- **优化逻辑**：
  - 用一个变量 `ret` 动态保存当前阶乘值。
  - 外层循环中，`ret` 初始为 1（1!），每轮更新 `ret = ret * n` 并累加到 `sum`。
- **优势**：
  - 时间复杂度从 O(n²) 降至 O(n)。
  - 代码更简洁高效。

## 在有序数组中查找指定元素（线性查找 vs 二分查找）

### 线性查找（基础方法）
- **实现**：遍历数组每个元素，与目标值比较。
- **缺点**：未利用“有序”特性，效率低（最坏情况需检查所有元素）。

### 二分查找（Binary Search）
- **前提**：数组必须**有序**。
- **核心思想**：每次比较中间元素，将搜索范围缩小一半。
- **步骤**：
  1. 初始化左边界 `left = 0`，右边界 `right = size - 1`。
  2. 循环条件：`left <= right`。
  3. 计算中点：`mid = (left + right) / 2`。
  4. 比较 `arr[mid]` 与目标值 `k`：
     - 若 `arr[mid] < k`：目标在右半区，`left = mid + 1`。
     - 若 `arr[mid] > k`：目标在左半区，`right = mid - 1`。
     - 若相等：找到目标，返回 `mid`。
  5. 循环结束未找到：返回“未找到”。

- **效率分析**：
  - 时间复杂度：O(log₂N)。
  - 示例：10 个元素最多查 4 次；42 亿元素最多查 32 次。
- **关键细节**：
  - 循环条件必须是 `left <= right`（包含单元素情况）。
  - 中点计算：`(left + right) / 2`（整数除法自动向下取整）。
  - 找到后需 `break` 退出循环。
