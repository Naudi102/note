## 内存的基本概念

- 计算机中的所有程序运行都在**内存（Memory）**中进行。
- 内存是程序运行所需的环境或空间，程序必须加载到内存中才能执行。
- 任务管理器可以查看每个程序占用的内存情况（如 VS2019、Chrome 等）。

## 内存单元与地址

- 为了高效管理内存，计算机将内存划分为一个个**内存单元**。
- 每个内存单元的大小为 **1 字节（Byte）**。
- 每个内存单元都有一个唯一的**编号**，这个编号称为**地址（Address）**。
- 地址的作用类似于现实生活中的房间号，便于快速定位和访问。

> 💡 **类比**：就像一栋楼里的每个房间都有编号（如 305），通过编号能快速找到具体房间。内存中的每个字节也有地址，通过地址可快速访问数据。

## 32位与64位系统下的内存管理

- **32位系统**：
  - 地址总线宽度为 32 位。
  - 可寻址空间为 $2^{32}$ 个地址。
  - 每个地址对应 1 字节 → 最大可管理内存为 $2^{32} \text{ Bytes} = 4 \text{ GB}$。
- **64位系统**：
  - 地址总线宽度为 64 位。
  - 理论最大内存远超当前硬件需求（实际常见 8GB、16GB、32GB）。
  - 指针大小为 8 字节（见后文）。

> ✅ **单位换算**：
> - 1 Byte (B) = 8 bits (b)
> - 1 KB = 1024 B
> - 1 MB = 1024 KB
> - 1 GB = 1024 MB

> 📌 **趣闻**：公司“字节跳动（ByteDance）”名称来源于“Byte”（字节），体现其技术背景。

## 变量与内存地址的关系

- 定义变量时，编译器会为其在内存中分配空间。
  ```c
  int a = 10;
  ```
  - 此语句在内存中申请 **4 字节**（`int` 类型通常占 4 字节）。
  - 这 4 字节连续存储，每个字节都有独立地址。
- **变量的地址**指的是其**首字节的地址**（即最小地址）。

（内存窗口显示地址 `0x00E0FF808` 开始的 4 字节内容，验证数据存储）

> 🔍 **注意**：整数 `10` 在内存中以**小端序（Little Endian）**存储（低位字节在前），因此看到的是倒序字节。

- 使用 **取地址操作符 `&`** 获取变量地址：
  ```c
  printf("%p\n", &a); // 打印 a 的地址，格式符为 %p
  ```
  - `%p` 用于打印指针（地址），通常以十六进制显示（更简洁，相比二进制）。

## 指针变量的定义与使用

- **指针（Pointer）的本质就是地址**。
- **指针变量**是用来**存储地址**的变量。

### 定义语法
```c
type *pointer_name;
```
- `type`：指针所指向对象的数据类型。
- `*`：表示该变量是指针。
- 示例：
  ```c
  int a = 10;
  int *p = &a;   // p 是一个指向 int 类型的指针，存储 a 的地址
  char ch = 'A';
  char *pc = &ch; // pc 是一个指向 char 类型的指针
  ```

### 关键理解
- `int *p` 中：
  - `*` 表明 `p` 是**指针变量**。
  - `int` 表明 `p` 指向的对象是 **`int` 类型**。
- 指针变量本身也是一个变量，有类型、有地址、有值（值是另一个变量的地址）。

> ⚠️ **常见误区**：不要认为 `int*` 是一个整体类型名。应理解为 “`p` 是一个指针，指向 `int`”。

## 解引用操作符

- 使用 **解引用操作符 `*`** 通过指针访问其所指向的变量。
  ```c
  *p = 20; // 等价于 a = 20;
  ```
- `*p` 表示“**p 所指向的那个对象**”。

### 操作符配对
- `&`（取地址）和 `*`（解引用）是一对互逆操作：
  ```c
  int a = 10;
  int *p = &a;
  *p = 20; // 修改 a 的值
  ```
  - `&a` 获取 `a` 的地址。
  - `*p` 通过地址访问 `a`。
  - 因此 `*&a` 等价于 `a`。

> ✅ **核心意义**：指针允许**间接访问和修改变量**，这是 C 语言强大功能（如函数传参、动态内存）的基础。

## 指针的大小

- 指针变量的大小**与其指向的类型无关**，只取决于**系统架构**：
  - **32位系统**：指针大小为 **4 字节**。
  - **64位系统**：指针大小为 **8 字节**。

### 验证代码
```c
#include <stdio.h>
int main() {
    printf("%zu\n", sizeof(int*));   // 输出 4 或 8
    printf("%zu\n", sizeof(char*));  // 输出 4 或 8
    printf("%zu\n", sizeof(double*)); // 输出 4 或 8
    return 0;
}
```

> 📌 **原因**：指针存储的是地址，而地址的大小由 CPU 地址总线宽度决定（32位 → 4B，64位 → 8B）。

> 💡 **提示**：在 Visual Studio 中，默认编译为 **32位（x86）**，故指针大小为 4 字节。若需测试 64 位，需切换平台为 x64。
