## 输入缓冲区与 scanf、getchar 的交互机制

视频深入讲解了 C 语言中标准输入函数（如 `scanf` 和 `getchar`）如何与**输入缓冲区（input buffer）**交互，这是理解用户输入行为的关键。

- 当用户在键盘上输入一串字符并按下回车（Enter）时，所有字符（包括空格、换行符 `\n`）都会被暂存到**输入缓冲区**中。
- `scanf` 在读取格式化输入（如 `%s`）时，**遇到空白字符（空格、制表符、换车符）即停止读取**，但**不会从缓冲区中移除这些空白字符之后的内容**。
  - 例如：输入 `AVCDF DFGHR\n`，`scanf("%s", str)` 只读取 `"AVCDF"`，缓冲区中仍保留 `" DFGHR\n"`。
- 紧接着调用 `getchar()` 会读取缓冲区中的**下一个字符**（通常是空格或 `\n`），而非等待新输入。

## 清除输入缓冲区的正确方法

为避免残留字符干扰后续输入（如菜单选择 Y/N），必须**清空缓冲区中 `scanf` 未处理的部分**。

### 错误做法
仅调用一次 `getchar()`：
```c
scanf("%s", password);
getchar(); // 仅能清除一个字符（如空格），无法处理后续内容
```
若输入含多个多余字符（如 `pwd extra\n`），此方法失效。

### 正确做法：使用 while 循环持续读取直至换行符
```c
scanf("%s", password);
// 清空缓冲区：读取并丢弃所有字符，直到遇到 '\n'
int ch;
while ((ch = getchar()) != '\n' && ch != EOF) {
    ; // 什么都不做，仅消耗字符
}
```
- **原理**：循环读取缓冲区中剩余字符，直到读到换行符 `\n`（表示用户本次输入结束）。
- **效果**：确保下一次输入（如 `getchar()` 读 Y/N）能正确等待用户新输入，而非读取残留字符。

## while 循环结合 getchar 实现字符过滤

演示了一个典型应用场景：**只输出数字字符，跳过其他字符**。

### 示例代码逻辑
```c
int ch;
while ((ch = getchar()) != EOF) {
    if (ch < '0' || ch > '9') { // 若不是数字字符
        continue; // 跳过本次循环，不执行后续输出
    }
    putchar(ch); // 输出数字字符
}
```

### 关键点解析
- **ASCII 值范围**：数字字符 `'0'` 到 `'9'` 的 ASCII 值连续（48–57）。
- **条件判断**：`ch < '0' || ch > '9'` 等价于 `!(ch >= '0' && ch <= '9')`。
- **`continue` 作用**：当读取到非数字字符（如字母、空格）时，立即跳过 `putchar`，继续下一次读取。
- **输入示例**：
  - 输入：`ABCD E234 WER\n`
  - 输出：`234`（仅数字被输出）

此模式展示了 `while` + `getchar` + `continue` 在**输入流过滤**中的强大应用。

## scanf 与 printf 在实际开发中的使用建议

讲师强调：**`scanf`/`printf` 主要用于学习和调试，非工业级软件开发的标准输入输出方式**。

- **现实软件架构**：
  - 用户通过**图形界面（GUI）或 Web 前端**输入数据。
  - 前端将数据传递给**后端服务**（可能用 C/C++ 编写）。
  - 后端处理完成后，将结果返回前端展示。
- **控制台程序局限性**：
  - 黑框框（命令行）交互体验差，不适合现代应用（如微信、浏览器）。
  - `scanf` 易受缓冲区问题困扰，健壮性不足。
- **学习阶段价值**：
  - 快速验证算法逻辑。
  - 理解输入/输出底层机制（如缓冲区）。
- **建议**：掌握其原理即可，工作中优先使用更安全的输入方法（如 `fgets`）或框架提供的 I/O 接口。

## 代码管理与 Git 提交实践

讲师分享了日常代码管理的最佳实践，鼓励学员使用 Git 记录学习过程。

### 推荐流程
1. **创建专属仓库**：如 `10607-c-language`，用于存放所有课程代码。
2. **按日期组织文件**：
   - 源码文件命名：`20240412_task.c`
   - 笔记文件命名：`20240412_notes.md`
3. **每日提交**：
   - 将当天工程文件（含 `.c`, `.sln`, `.vcxproj` 等）整体提交。
   - 提交信息清晰：如 “提交 4月12日 代码与笔记”。
4. **推送至远程**：使用 `git push` 同步到 GitHub/Gitee，便于复习和展示。

### 重要观念
- **不要羞于提交“不完美”代码**：仓库是个人学习历程的记录，非生产项目。
- **避免频繁重命名**：固定文件名（如 `task.c`）会导致历史混乱，建议包含日期。
- **配置文件非垃圾**：`.sln`、`.vcxproj` 等是 Visual Studio 工程必需文件，应一并提交。

## 常见问题与注意事项

### 关于换行符（`\n`）
- **回车 ≠ 单个字符**：在 Windows 中，按 Enter 实际产生两个字符 `\r\n`（Carriage Return + Line Feed），但 C 运行时通常将其转换为单个 `\n`。
- **密码中能否包含换行？**  
  理论上可以，但 `scanf("%s")` 无法读取含换行的字符串（因遇 `\n` 停止）。若需读取含空白的密码，应使用 `fgets()`。

### Visual Studio 调试小技巧
- **程序不退出？**：VS 有时需按 **三次 Enter** 才能结束 `getchar()` 循环，属 IDE 特性，非代码问题。其他编译器（如 GCC）通常一次即可。

### getchar 返回值类型
- **务必用 `int` 接收**：`getchar()` 返回 `int` 类型，以容纳 `EOF`（通常为 -1）。若用 `char`，可能无法正确识别 `EOF`。
