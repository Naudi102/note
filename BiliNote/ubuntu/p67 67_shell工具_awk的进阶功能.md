## AWK 内置变量概述

AWK 除了支持用户自定义变量外，还提供了一系列**系统内置变量**，无需声明即可直接使用。这些变量在处理文本时非常实用，尤其适用于日志分析、数据提取等场景。

> **注意**：在 AWK 命令中，**不加双引号的标识符会被视为内置变量**；若加了双引号，则被当作字符串字面量。

## 常用内置变量详解

以下为几个最常用的 AWK 内置变量：

- `FILENAME`：当前正在处理的文件名。
- `NR`（Number of Record）：当前记录（行）的编号，从 1 开始计数，**跨文件连续递增**。
- `NF`（Number of Fields）：当前行的字段（列）数量。

> 视频中口误将 `NF` 说成 “NRF”，实际应为 `NF`。

## 实战应用示例

### 显示文件名、行号与列数

通过组合内置变量，可一次性输出每行的文件名、行号和列数：

```bash
awk -F';' '{print "文件名:", FILENAME, "行号:", NR, "列数:", NF}' data.txt
```

- `-F';'`：指定字段分隔符为分号。
- 单引号内直接使用 `FILENAME`、`NR`、`NF`，无需引号。
- 若在 `print` 中使用双引号包裹内容（如 `"行号:"`），则为字符串；变量名不加引号才被识别为内置变量。

### 查找空行及其行号

空行的特征是：**没有字段**，即 `NF == 0`。

命令如下：

```bash
some_command | awk 'NF == 0 {print NR}'
```

- `NF == 0` 作为匹配条件，仅当某行字段数为 0 时执行 `{print NR}`。
- 此方法可快速定位空行位置，适用于清理日志或配置文件。

> 视频强调：条件表达式必须使用**正斜杠 `/` 的逻辑写法（即直接写条件）**，而非正则表达式斜杠。此处“钢”应为“刚”或指“条件写法”，实际应写作 `NF == 0`，不能写成 `/NF == 0/`。

### 使用 AWK 提取 IP 地址

目标：从 `ip addr show` 或类似命令输出中提取 IPv4 地址（通常出现在包含 `inet` 的行中）。

传统方法需结合 `grep` + `awk`，但 AWK 可独立完成：

```bash
ip addr show | awk '/inet/ {print $2}'
```

- `/inet/`：作为模式，匹配包含 “inet” 的行。
- `$2`：打印该行的第 2 个字段（即 IP 地址/子网，如 `192.168.1.10/24`）。

> **关键细节**：AWK 默认以**空白字符（空格或制表符）为分隔符**，并**自动忽略空行和前导/尾随空格**。因此，即使原始输出有缩进，`$1` 仍为第一个非空字段（如 `inet`），IP 地址通常在 `$2`。

视频演示发现：原以为 IP 在第 10 列，实测后确认在第 2 列，凸显 AWK 智能分词能力。

## 关键语法与注意事项

- **单引号包裹 AWK 程序**：防止 shell 解释变量或特殊字符。
- **变量 vs 字符串**：
  - `print NR` → 输出行号（变量）
  - `print "NR"` → 输出字符串 "NR"
- **字段分隔符**：默认为空白，可用 `-F` 指定（如 `-F','`）。
- **模式匹配**：`/pattern/ { action }` 是标准写法，`pattern` 可为正则或条件（如 `NF == 0`）。
- **AWK 自动跳过空行**：在计算字段时，空行 `NF = 0`，不会干扰其他行的字段编号。
