## PS 命令概述

`ps`（Process Status）是 Linux 中用于查看当前系统进程状态的核心命令。它能显示正在运行的程序（即进程）及其资源占用情况。

- 默认执行 `ps` 仅显示当前终端中运行的前台进程（通常只有 shell 和 `ps` 自身）。
- 要查看所有进程，需使用扩展选项：
  - `ps aux`
  - `ps -ef`

## PS aux 详解

`ps aux` 是最常用的查看所有进程的方式，输出包含 11 列信息：

| 列序 | 字段名 | 含义说明 |
|------|--------|----------|
| 1 | USER | 进程的创建者（如 `atguigu`） |
| 2 | PID | 进程唯一标识号（Process ID），系统重启前不会重复，只会递增 |
| 3 | %CPU | CPU 使用率（百分比）。0% 表示当前未占用 CPU 时间片（因 CPU 调度快速切换） |
| 4 | %MEM | 内存使用率（百分比）。极小值可能显示为 0.0 |
| 5 | VSZ | 虚拟内存大小（单位 KB） |
| 6 | RSS | 实际物理内存占用（Resident Set Size，单位 KB） |
| 7 | TTY | 进程关联的终端设备。常见值：<br>• `TTY1–TTY6`：本地文本终端<br>• `TTY7`：图形界面终端<br>• `pts/0–255`：伪终端（如 SSH 连接）<br>• `?`：无关联终端（常用于后台服务） |
| 8 | STAT | 进程状态：<br>• `R`：运行中（Running）<br>• `S`：睡眠（Sleeping，可中断）<br>• `T`：停止（Stopped）<br>• `+`：位于前台进程组<br>• `s`：会话领导者（session leader） |
| 9 | START | 进程启动时间（如 `18:23`） |
| 10 | TIME | 进程累计占用 CPU 的总时间（如 `00:01` 表示 1 秒） |
| 11 | COMMAND | 启动该进程的完整命令（如 `/usr/sbin/sshd`） |

> 示例：通过 `ps aux | grep 26992` 可过滤特定 PID 的进程（注意第二行是 `grep` 自身，应忽略）。

## PS -ef 详解

`ps -ef` 侧重展示**进程间的父子关系**（树状结构），适用于分析进程依赖或批量终止。

输出字段说明：

| 列序 | 字段名 | 含义说明 |
|------|--------|----------|
| 1 | UID | 用户 ID（等同于 USER） |
| 2 | PID | 当前进程 ID |
| 3 | PPID | 父进程 ID（Parent PID）<br>• 若 PPID = 1，表示由 init/systemd 直接启动（系统级服务）<br>• 父进程终止时，其子进程通常也会被终止 |
| 4 | C | CPU 调度优先级（旧式字段，现代系统意义有限） |
| 5 | STIME | 进程启动时间 |
| 6 | TTY | 同 `ps aux` 中的 TTY |
| 7 | TIME | 累计 CPU 使用时间 |
| 8 | CMD | 启动命令 |

> 示例：SSH 连接（PID 26992）可能派生多个子进程（如 `sftp-server`），通过 `-ef` 可清晰看到其 PPID 指向主 sshd 进程。

## aux 与 -ef 的使用场景对比

| 使用场景 | 推荐命令 | 原因 |
|---------|----------|------|
| 查看 CPU/内存资源占用 | `ps aux` | 包含 `%CPU`、`%MEM`、`RSS` 等详细资源指标 |
| 分析进程依赖关系 | `ps -ef` | 明确显示 `PPID`，便于理解父子进程结构 |
| 终止进程（尤其是服务） | `ps -ef` | 避免误杀父进程导致连锁终止；可先确认依赖树 |
| 快速定位某用户进程 | `ps aux` | USER 列直观，配合 `grep` 过滤高效 |

> **关键建议**：  
> - 若需杀死进程，务必先用 `ps -ef` 确认其父进程，防止意外终止关键服务。  
> - 系统核心进程（PPID=1）通常不应手动终止，否则可能导致系统不稳定。

## AI 总结

本节详细讲解了 Linux 中 `ps` 命令的两种核心用法：`ps aux` 侧重资源监控（CPU、内存），适合性能分析；`ps -ef` 侧重进程树结构，适合管理与排错。理解各列含义（如 PID、PPID、STAT、%CPU）是掌握系统监控的基础。在实际运维中，应根据目标选择合适命令，并结合 `grep` 过滤提高效率。
