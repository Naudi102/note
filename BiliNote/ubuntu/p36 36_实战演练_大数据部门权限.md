## 实战目标

本节实战演练模拟 **大数据部门（BakeData）** 的权限管理需求，具体要求如下：

- 部门领导（王总）拥有 **完全控制权限（rwx）**。
- 部门成员（如小王）拥有 **读、写、执行权限（rwx）**。
- **非部门人员** 对该目录及内部文件 **无任何权限（---）**。
- 所有操作在 `/opt/BakeData` 目录下进行。

## 用户与组创建

### 1. 创建部门领导账号（王总）
```bash
sudo useradd 王总
# 设置密码为 123456
sudo passwd 王总
```

### 2. 创建普通成员账号（小王）
```bash
sudo useradd 小王
sudo passwd 小王  # 密码同样设为 123456
```

### 3. 创建部门组（BakeData）
```bash
sudo groupadd BakeData
```

### 4. 将用户加入部门组
- 将 **王总** 和 **小王** 加入 `BakeData` 组：
```bash
sudo usermod -aG BakeData 王总
sudo usermod -aG BakeData 小王
```

> ⚠️ 注意：`usermod -aG` 中的 `-a`（append）参数非常重要，避免覆盖用户原有组。

## 工作目录设置

### 1. 创建工作目录
```bash
sudo mkdir /opt/BakeData
```

### 2. 设置目录所有者和所属组
```bash
sudo chown 王总:BakeData /opt/BakeData
```

> 此操作确保目录由王总拥有，且所属组为 `BakeData`。

## 权限配置

### 1. 设置目录权限为 `770`
```bash
chmod 770 /opt/BakeData
```

- **第一位 `7`**：所有者（王总）→ rwx
- **第二位 `7`**：所属组（BakeData 成员）→ rwx
- **第三位 `0`**：其他用户 → 无权限

> ✅ 满足“部门内可读写执行，外部完全隔离”的安全要求。

## 文件创建与权限测试

### 1. 王总创建可执行文件
以王总身份登录后：
```bash
cd /opt/BakeData
vim BakeData.txt
# 内容：Hello BakeData
# 保存退出
chmod 770 BakeData.txt
sudo chown 王总:BakeData BakeData.txt
```

### 2. 小王创建文件并设置权限
以小王身份登录后：
```bash
cd /opt/BakeData
vim 小王.trt
# 内容：Hello 王总
# 保存退出
```

#### 关键步骤：修改新文件的所属组
```bash
sudo chown 小王:BakeData 小王.trt
chmod 770 小王.trt
```

> ❗ 若不修改所属组，默认组为 `小王`，导致王总无法访问（被视为“其他用户”）。

### 3. 权限验证
- 王总可成功执行 `./小王.trt`
- 非 `BakeData` 组用户无法进入 `/opt/BakeData` 或读取其中文件

## 常见问题与解决方案

### 问题1：王总无法访问小王创建的文件
- **原因**：王总未被加入 `BakeData` 组（初始遗漏）
- **解决**：
  ```bash
  sudo usermod -aG BakeData 王总
  ```
  > ⚠️ 修改组后需 **重新登录终端** 才能生效（组信息缓存）

### 问题2：新创建文件默认组不是 `BakeData`
- **原因**：用户主组（primary group）仍为自身（如 `小王`）
- **后果**：新文件属组为 `小王`，王总无法访问

## 最佳实践补充

### 推荐做法：将部门成员的主组设为 `BakeData`
```bash
sudo usermod -g BakeData 小王
```

- `-g`（小写）用于修改 **主组（primary group）**
- 设置后，小王新建的所有文件默认属组为 `BakeData`
- 无需每次手动 `chown` 修改组

> ✅ 此方法从根本上避免权限错配问题，提升团队协作效率。

### 补充规则
- **只有 root 或文件所有者** 可修改文件的 **所有者（owner）**
- **修改文件所属组（group）** 需满足：
  1. 操作者是文件所有者 **或** root
  2. 操作者 **同时属于原组和目标组**

## AI 总结

本节通过模拟大数据部门权限管理场景，系统演示了 Linux 用户、组、目录及文件权限的精细化配置。核心要点包括：使用 `770` 权限实现部门内全权限、外部零访问；通过 `usermod -aG` 正确管理附加组；强调主组（`-g`）设置对自动继承权限的重要性；并揭示了权限变更后需重新登录才能生效的机制。该案例为实际企业级权限隔离提供了可复用的最佳实践模板。
