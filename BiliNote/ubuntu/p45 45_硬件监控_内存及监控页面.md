## 内存使用情况查看命令：free -m

- **命令**：`free -m`
  - 用于查看服务器内存的使用情况。
  - `-m` 参数表示以 **MB（兆字节）** 为单位显示，比默认的 KB 更直观。
  - 若不加 `-m`，单位为 KB，数值较大不易阅读。

- **输出字段说明**：
  - **Mem（物理内存）**：重点关注的部分。
    - `total`：总内存容量。
    - `used`：已使用内存。
    - `free`：完全未使用的内存。
    - `available`：可用内存（包括可回收的缓存/缓冲区），更真实反映系统可分配内存。
  - **Swap（交换内存）**：虚拟内存，通常在物理内存不足时使用，一般只需关注物理内存即可。

> 建议日常使用 `free -m` 快速判断系统内存是否充足。

## 系统实时监控命令：top

- **命令**：`top`
  - 实时动态监控系统资源使用情况。
  - 自动刷新界面，默认按 CPU 使用率排序，高占用进程会自动置顶。

- **顶部信息区域解析**：
  - **系统时间**：当前时间。
  - **Uptime**：系统已运行时间（不包含挂起时间）。
  - **Users**：当前登录用户数。
  - **Load average**：系统负载（1分钟、5分钟、15分钟平均值），值越低表示系统越空闲。
  - **Tasks**：进程状态统计：
    - running（运行中）
    - sleeping（休眠）
    - stopped（已停止）
    - zombie（僵尸进程，通常需排查）

- **内存与交换区信息**：
  - 整合显示物理内存（Mem）和交换空间（Swap）的使用情况，与 `free` 命令类似。

- **进程列表区域**：
  - 类似 `ps` 命令的输出，包含 PID、USER、%CPU、%MEM、COMMAND 等字段。
  - 可通过交互命令（如 `k` 杀进程、`q` 退出）操作。

> 虽然 `top` 是历史悠久的经典工具，但界面较原始，适合快速排查，不推荐长期监控。

## 图形化监控工具：Ubuntu 系统监视器

- **推荐工具**：Ubuntu 自带的 **系统监视器（System Monitor）**
  - 图形界面，操作直观，适合日常使用。
  - 功能全面，包含多个标签页：
    - **进程（Processes）**：查看所有运行进程，支持排序、结束任务。
    - **资源（Resources）**：
      - CPU 使用率（按核心显示，支持多核视图）
      - 内存与交换空间使用趋势图
      - 网络吞吐量（接收/发送）
      - 磁盘 I/O 活动
    - **文件系统（File Systems）**：挂载点、磁盘使用情况（类似 `df` 命令）

- **优势**：
  - 实时动态图表，可视化程度高。
  - 支持长时间后台运行，便于持续监控。
  - 无需记忆命令，适合初学者和非运维人员。

> 对于大多数用户，尤其是开发者和嵌入式学习者，优先使用系统监视器比命令行工具更高效。

## 实用建议与总结

- **命令行工具适用场景**：
  - 服务器无图形界面环境。
  - 需要快速诊断（如 SSH 远程登录时）。
  - 常用命令：`free -m`（内存）、`df -h`（磁盘）、`ps`（进程）、`top`（综合监控）。

- **图形工具适用场景**：
  - 本地开发机或桌面版 Ubuntu。
  - 需要长期、直观地观察系统性能。
  - 推荐保持“系统监视器”常开，便于及时发现异常。

- **关键结论**：
  - 不必深入研究 `top` 的古老界面，掌握基础命令即可。
  - 现代 Linux 系统已有成熟 GUI 工具，应善用之。
  - 初期硬件资源充足，重点在于快速定位问题，而非复杂监控。
