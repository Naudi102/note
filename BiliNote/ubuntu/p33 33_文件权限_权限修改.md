## 文件权限基础回顾

- Linux 中每个文件/目录的权限分为三组：
  - **User (u)**：文件所有者
  - **Group (g)**：所属用户组
  - **Others (o)**：其他用户（视频中口误为 “Azir”，应为 “Others”）
- 每组权限包含三种基本权限：
  - **r (read)**：读取
  - **w (write)**：写入
  - **x (execute)**：执行
- 权限格式统一适用于文件和目录。

## 使用符号方式修改权限（chmod + 符号）

### 基本语法
```bash
chmod [who][operator][permission] 文件/目录
```
- **who**:
  - `u`：用户（所有者）
  - `g`：组
  - `o`：其他用户
  - `a`（或省略）：全部（all）
- **operator**:
  - `+`：添加权限
  - `-`：移除权限
- **permission**:
  - `r`, `w`, `x`

### 示例操作

1. **为所有者添加执行权限**：
   ```bash
   chmod u+x Purple
   ```
   - 执行后，所有者可运行该文件（前提是文件内容本身可执行）。
   - *注意：若文件非脚本或二进制程序，即使有 x 权限也会报错。*

2. **为所属组添加读写权限**：
   ```bash
   chmod g+rw Purple
   ```

3. **为所有用户添加执行权限**（`a` 可省略）：
   ```bash
   chmod +x Purple
   # 等价于 chmod a+x Purple
   ```

### 局限性
- 若需为三组分别设置不同权限（如 u=rwx, g=rw, o=r），需多次执行命令：
  ```bash
  chmod g-wx Purple    # 移除组的 w 和 x
  chmod o-wx Purple    # 移除其他的 w 和 x
  ```
- 操作繁琐，效率低。

## 使用数字方式修改权限（chmod + 数字）

### 原理
- 每组权限用一个 **三位二进制数** 表示，对应 rwx：
  - r = 4（100）
  - w = 2（010）
  - x = 1（001）
- 将所需权限对应的数值相加，得到该组的权限数字。

| 权限组合 | 二进制 | 十进制 |
|----------|--------|--------|
| rwx      | 111    | 7      |
| rw-      | 110    | 6      |
| r-x      | 101    | 5      |
| r--      | 100    | 4      |
| -wx      | 011    | 3      |
| --x      | 001    | 1      |
| ---      | 000    | 0      |

### 示例：设置权限为 764
- **User**: rwx → 4+2+1 = **7**
- **Group**: rw- → 4+2 = **6**
- **Others**: r-- → 4 = **4**

```bash
chmod 764 group.txt
```
- 一步到位完成复杂权限设置。

### 适用性
- 同样适用于**文件和目录**。
- 是**更专业、更常用**的权限设置方式。

## 关键对比与建议

| 方式       | 优点                     | 缺点                     | 适用场景               |
|------------|--------------------------|--------------------------|------------------------|
| 符号方式   | 直观、易理解             | 多组权限需多次命令       | 临时微调单一权限       |
| 数字方式   | 一步设置完整权限，高效   | 需记忆数字含义           | 精确配置、脚本自动化   |

> **建议**：日常使用中优先掌握数字方式（如 755、644、764 等常见组合），提升效率。

## AI 总结

本节详细讲解了 Linux（Ubuntu）系统中文件与目录权限的两种修改方法：符号法（如 `chmod u+x`）和数字法（如 `chmod 764`）。符号法直观但操作繁琐，适合简单调整；数字法基于二进制位权（r=4, w=2, x=1），能一次性精确设置三组权限，是专业开发和系统管理中的主流方式。理解并熟练运用数字权限表示法，对提升 Linux 操作效率至关重要。
