## 删除文件/目录命令：`rm`

- `rm`（remove）命令用于**彻底删除**文件或目录，**不会进入回收站**。
- 默认情况下，`rm` **只能删除文件**，不能直接删除目录。
- 要删除目录，需使用选项：
  - `-r`（recursive）：递归删除整个目录及其内容。
  - `-f`（force）：强制删除，不提示确认。
  - 常用组合：`rm -rf 目录名` → **彻底、无提示、递归删除**。
- 示例：
  ```bash
  rm 文件名          # 删除单个文件
  rm -r 目录名       # 递归删除目录（会提示确认）
  rm -rf 目录名      # 强制彻底删除目录（无确认）
  ```
- ⚠️ **警告**：`rm` 操作不可逆，删除后无法从回收站恢复。

## 移动与重命名命令：`mv`

- `mv`（move）命令用于：
  - **移动文件/目录**到另一个位置。
  - **重命名**文件或目录（当源路径与目标路径在同一目录时）。
- 语法：
  ```bash
  mv 源路径 目标路径
  ```
- 示例：
  - 移动文件：
    ```bash
    mv ~/桌面/吸油器 ~/tmp/    # 将“吸油器”移动到 tmp 目录
    ```
  - 重命名：
    ```bash
    mv ~/桌面/吸油器 ~/桌面/大化吸油    # 重命名为“大化吸油”
    ```
- `mv` 可直接操作目录，无需额外参数。

## 回收站机制与图形界面操作

- **图形界面（GUI）中的删除**（如按 Delete 键）：
  - 文件会被移入 **回收站（Trash）**，可恢复。
  - 清空回收站后才真正删除。
- **终端中无法直接将文件移入系统回收站**：
  - 尝试 `mv 文件 ~/.local/share/Trash/` 会失败或不被识别为有效回收操作。
  - 系统回收站是图形环境的专属功能，命令行不原生支持。
- **替代方案**：创建自定义“临时回收目录”（如 `~/tmp`），用 `mv` 将文件移入，模拟回收站行为。
  ```bash
  mkdir -p ~/tmp
  mv 不再需要的文件 ~/tmp/
  ```

## 安全建议与最佳实践

- **避免频繁使用 `rm -rf`**，尤其在不确定路径时。
- 删除前建议：
  - 先用 `ls` 确认目标是否存在。
  - 对重要数据先备份或移至临时目录。
- 日常操作推荐使用 `mv` + 自定义临时目录，而非直接 `rm`。
- 若需图形化回收站功能，请在桌面环境中操作（如右键删除或按 Delete 键）。
