## read 命令概述

在 Shell 脚本中，`read` 是一个用于**从标准输入（通常是终端）读取用户输入**的内置命令。它常用于实现交互式脚本，例如在添加用户、配置设置或收集用户信息时。

- 用户输入的内容会被赋值给指定的变量。
- 若未提供输入（如超时或直接回车），变量将被设为空字符串。
- `read` 通常在脚本中使用，而非直接在命令行交互中频繁调用。

## read 命令基本语法与参数

### 基本格式：
```bash
read [选项] 变量名
```

### 常用选项：

- `-p "提示信息"`  
  在读取输入前显示提示文本（支持中文等字符）。  
  示例：`read -p "请输入你的名称: " name`

- `-t 超时秒数`  
  设置等待用户输入的最长时间（单位：秒）。若超时未输入，`read` 返回非零状态，变量为空。  
  示例：`read -t 7 -p "请在7秒内输入: " input`

> 注意：选项顺序不影响功能，但习惯上先写 `-t` 再写 `-p`。

### 输入行为说明：
- 用户输入后按 **Enter** 键确认。
- 若在 `-t` 指定时间内无输入或直接回车，变量值为空。
- 变量名必须合法（如 `username`、`name`），不能包含空格或特殊符号。

## 实战示例：带提示与超时的 read 脚本

### 脚本内容（`read.sh`）：
```bash
#!/bin/bash
read -t 7 -p "请在7秒内输入你的名称: " name
echo "你好，$name"
```

### 创建与执行步骤：
1. 创建脚本文件：
   ```bash
   vim read.sh
   ```
2. 写入上述内容并保存。
3. 添加可执行权限：
   ```bash
   chmod +x read.sh
   ```
4. 运行脚本：
   ```bash
   ./read.sh
   ```

### 预期行为：
- 终端显示提示：“请在7秒内输入你的名称: ”
- 用户在7秒内输入（如 `CRK`）并回车 → 输出 `你好，CRK`
- 用户超过7秒未输入 → 自动继续执行，输出 `你好，`（变量为空）

## 执行效果演示

- **正常输入场景**：
  ```
  请在7秒内输入你的名称: CRK
  你好，CRK
  ```

- **超时场景**：
  （等待7秒无输入）
  ```
  请在7秒内输入你的名称: 
  你好，
  ```

此机制非常适合需要限时响应的自动化脚本，避免程序无限挂起。

## AI 总结

`read` 命令是 Bash 脚本中实现用户交互的核心工具，通过 `-p` 提供友好提示，结合 `-t` 设置超时机制，可有效提升脚本的健壮性与用户体验。在实际开发中，建议始终为 `read` 添加超时限制，防止脚本因等待输入而卡死。本节内容为后续编写交互式 Shell 脚本奠定了重要基础。
