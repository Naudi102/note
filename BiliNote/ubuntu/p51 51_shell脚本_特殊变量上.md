## 特殊变量概述

在 Shell 脚本中，除了自定义变量外，还有一类**特殊变量（Special Variables）**，用于实现特定功能，尤其在处理**外部传参**时非常关键。这些变量由 Shell 自动设置，无需手动赋值。

本节重点介绍两个最常用的特殊变量：
- `$N`（如 `$1`, `$2`, ..., `${10}`）：表示传递给脚本的第 N 个参数
- `$#`：表示传递给脚本的参数总个数

---

## $N：位置参数

### 基本概念
- `$0`：表示**脚本本身的名称**（不是第一个参数）
- `$1`：第一个外部参数
- `$2`：第二个外部参数
- ...
- `$9`：第九个外部参数
- `${10}`、`${11}`...：第十个及以后的参数（**必须加花括号 `{}`**）

> ⚠️ **重要规则**：当参数序号 ≥ 10 时，必须使用 `${10}` 格式。若写成 `$10`，Shell 会将其解释为 `$1` 后跟字符 `0`，导致错误。

### 示例演示
创建脚本 `Parameters.sh`：
```bash
#!/bin/bash
echo "脚本名 (\$0): $0"
echo "参数1 (\$1): $1"
echo "参数2 (\$2): $2"
echo "参数10 (\${10}): ${10}"
```

执行命令：
```bash
bash Parameters.sh hello world 3 4 5 6 7 8 9 10
```

输出结果：
```
脚本名 ($0): Parameters.sh
参数1 ($1): hello
参数2 ($2): world
参数10 (${10}): 10
```

*![](http://localhost:8000/static/screenshots/screenshot_034bfe22c-fa5f-4460-a3c1-164d00a1bc77.jpg)*  
（展示脚本执行后 `$0` 显示脚本名，`$1`~`$2` 显示对应参数，`${10}` 正确获取第十个参数）

### 应用场景
- 类似 `ls -l` 中的 `-l` 就是传递给 `ls` 命令的参数
- 自定义脚本可通过 `$1`, `$2` 等接收用户输入的选项或文件路径
- 实际命令（如 `cp src dest`）通过解析 `$1`（源文件）和 `$2`（目标文件）来区分参数含义

> 💡 虽然原理相同，但成熟命令通常会进行更复杂的参数解析（如判断是否以 `-` 开头、参数顺序、合法性校验等）。

## $#：参数个数

### 基本概念
- `$#`：返回传递给脚本的**位置参数的总数量**（不包括 `$0`）
- 常用于根据参数数量执行不同逻辑（例如：参数不足时提示用法）

### 示例演示
在 `Parameters.sh` 中添加：
```bash
echo "------------------"
echo "参数个数 (\$#): $#"
```

执行测试：
```bash
bash Parameters.sh a          # 输出：1
bash Parameters.sh a b c      # 输出：3
```

*![](http://localhost:8000/static/screenshots/screenshot_1299c4ee0-fb81-4bc9-ac2e-54d21e1a2071.jpg)*  
（展示不同参数数量下 `$#` 的输出值）

### 注意事项
- 在 `echo` 中使用 `$#` 时，建议用**双引号**包裹以确保正确解析：
  ```bash
  echo "参数个数: $#"
  ```
- 若使用单引号（如 `echo '参数个数: $#'`），则 `$#` 不会被解析，直接输出字面量 `$#`

*![](http://localhost:8000/static/screenshots/screenshot_21c6ca593-9ca6-44c3-9c96-54806c5c5aee.jpg)*  
（对比单引号与双引号下 `$#` 的输出差异）

### 实际用途
- 脚本开头可检查 `$#` 是否符合预期，避免因参数缺失导致错误
- 用于循环遍历所有参数（配合 `while` 或 `for`）

## AI 总结

本节介绍了 Shell 脚本中两个核心特殊变量：`$N` 和 `$#`。`$0` 表示脚本名，`$1`~`$9` 表示前九个参数，十位及以上参数需用 `${10}` 格式；`$#` 返回参数总数，常用于参数校验。这些变量是实现脚本外部传参的基础，广泛应用于命令行工具开发。掌握其用法对编写灵活、健壮的 Shell 脚本至关重要。
