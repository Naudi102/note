## 核心概念

- **grep**（视频中发音为“Grey”或“Greyport”）是 Linux/Ubuntu 中用于**文本行过滤**（即“过滤查找”）的命令。
- **grep 不能单独使用**，必须配合能产生标准输出（stdout）的命令，通过**管道符（`|`）** 将前一个命令的输出作为 grep 的输入。
- **管道符 `|`** 的作用是将前一个命令的输出“传递”给后一个命令处理，实现命令链式操作。

## 基本语法与使用方式

```bash
command | grep "搜索关键词"
```

- 示例：`ls -l | grep "maisan"`  
  表示：列出当前目录详细信息，并从中筛选出包含 “maisan” 的行。
- grep 默认进行**模糊匹配**（只要行中包含关键词即匹配）。
- 匹配成功的行会以**高亮颜色**显示关键词（取决于终端配色）。

## 常用参数详解

### `-n`：显示匹配行的行号
- 用法：`command | grep -n "keyword"`
- 示例：`cat Parfair | grep -n "AV"`  
  输出如：`470:...AV...`，表示第 470 行包含 “AV”。

### `-v`：反向匹配（排除匹配行）
- 用法：`command | grep -v "keyword"`
- 功能：**显示不包含关键词的所有行**。
- 示例：`ls -l | grep -v "maisan"`  
  将显示除包含 “maisan” 的所有其他行。

## 实际应用场景示例

1. **过滤文件列表**  
   ```bash
   ls -l | grep "DRW"
   ```
   快速找出权限字段包含 “DRW” 的文件/目录。

2. **在文件内容中搜索关键词**  
   ```bash
   cat ~/Desktop/Parfair | grep -n "AV"
   ```
   在名为 `Parfair` 的文件中查找所有含 “AV” 的行，并显示行号。

3. **在复杂命令输出中定位信息**  
   当命令输出内容过多时（如 `ps aux`、`systemctl list-units` 等），可使用：
   ```bash
   some_long_output_command | grep "RS"
   ```
   快速判断输出中是否包含目标信息（如 “RS”）。

## 关键注意事项

- **grep 始终按“整行”处理**：  
  - 如果某行包含关键词，则**整行输出**；  
  - 如果不包含，则**整行丢弃**。
- **必须位于管道符之后**：  
  grep 几乎没有独立使用的场景，必须依赖前序命令提供输入流。
- **支持正则表达式**（本视频未深入，但属高级用法）。
- **大小写敏感**：默认区分大小写，如需忽略可用 `-i` 参数（虽未提及，但属常见扩展）。
