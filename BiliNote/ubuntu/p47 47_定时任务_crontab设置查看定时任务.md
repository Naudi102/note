## crontab 简介

`crontab`（常被误读为 "coron table"）是 Linux/Ubuntu 系统中用于设置**定时任务**（scheduled tasks）的核心工具。它允许用户在指定时间自动执行命令或脚本，类似于“给系统设置闹钟”。

- 定时任务可用于自动化备份、日志清理、数据同步等场景。
- Ubuntu 默认已启用 `cron` 服务，无需手动启动。

## crontab 服务状态与基本命令

### 检查 cron 服务状态
```bash
systemctl status cron
```
- Ubuntu 默认已开启 `cron` 服务。

### crontab 常用命令选项
| 命令 | 功能 |
|------|------|
| `crontab -e` | 编辑当前用户的定时任务（进入编辑器） |
| `crontab -l` | 列出当前用户的所有定时任务 |
| `crontab -r` | 删除当前用户的所有定时任务 |

> **注意**：`crontab` 操作默认仅针对**当前用户**，不会影响其他用户。

## crontab 定时任务的用户隔离机制

- Ubuntu 是多用户操作系统，每个用户的 `crontab` 任务相互独立。
- 使用 `sudo crontab -l` 可查看 root 用户的任务，普通用户只能看到自己的任务。
- 执行 `crontab` 命令时，若未指定用户，默认操作当前登录用户。

## crontab 编辑环境配置

首次使用 `crontab -e` 时，若未设置默认编辑器，系统会提示选择或报错。

### 设置默认编辑器（如 vim）
```bash
export EDITOR=vim
```
- 设置后，再次运行 `crontab -e` 即可正常进入编辑界面。

## crontab 时间格式详解

每条 `crontab` 任务由 **6 个字段**组成，格式如下：

```
分 时 日 月 周 命令
```

| 字段 | 含义 | 取值范围 | 特殊符号 |
|------|------|----------|----------|
| 分（Minute） | 分钟 | 0–59 | `*`, `,`, `-`, `*/n` |
| 时（Hour） | 小时（24小时制） | 0–23 | 同上 |
| 日（Day of month） | 月份中的第几天 | 1–31 | 同上 |
| 月（Month） | 月份 | 1–12 | 同上 |
| 周（Day of week） | 星期几（0=周日） | 0–7（0和7都表示周日） | 同上 |
| 命令 | 要执行的命令或脚本 | — | — |

### 重要规则与注意事项

1. **“日”和“周”互斥**：
   - 如果指定了“周”（如 `1` 表示周一），则“日”字段应设为 `*`。
   - 反之，若指定了“日”（如 `15`），则“周”字段应设为 `*`。
   - **避免同时设置具体“日”和“周”**，否则逻辑混乱，可能导致任务不按预期执行。

2. **通配符与语法**：
   - `*`：表示任意值（如 `*` 在“分”字段表示每分钟）。
   - `,`：列举多个值（如 `0,30` 表示第0分和第30分）。
   - `-`：表示连续范围（如 `1-5` 表示1到5）。
   - `*/n`：表示每隔 n 个单位执行一次（如 `*/10` 表示每10分钟）。

### 常见示例

| crontab 行 | 含义 |
|-----------|------|
| `45 22 * * * command` | 每天 22:45 执行 |
| `0 17 * * 1 command` | 每周一 17:00 执行 |
| `0 5 1,15 * * command` | 每月1号和15号 5:00 执行 |
| `40 4 * * 1-5 command` | 周一至周五 4:40 执行 |
| `*/10 4 * * * command` | 每天 4 点内每10分钟执行一次（4:00, 4:10, ..., 4:50）|

> **警告**：不要同时设置具体的“日”和“周”，例如 `0 0 1 * 1 command` 是不推荐的写法。

## crontab 实战示例

### 目标
每分钟向桌面文件 `baolouma.trt` 写入一行 `11`。

### 步骤

1. **创建目标文件**（可选，crontab 会自动创建）：
   ```bash
   cd ~/Desktop
   touch baolouma.trt
   ```

2. **编辑 crontab**：
   ```bash
   crontab -e
   ```
   添加以下行：
   ```bash
   * * * * * echo "11" >> /home/用户名/Desktop/baolouma.trt
   ```
   > 注意：路径需使用**绝对路径**，且确保用户有写权限。

3. **验证任务是否添加成功**：
   ```bash
   crontab -l
   ```

4. **实时监控文件变化**：
   ```bash
   tail -f ~/Desktop/baolouma.trt
   ```
   - 每隔一分钟，文件将新增一行 `11`。

*![](http://localhost:8000/static/screenshots/screenshot_3d7cf417b-0b6a-4915-a123-b304dca8955a.jpg)*  
（显示 `tail -f` 输出中每隔一分钟新增一行 `11`）

### 关键点总结
- 命令必须使用**完整路径**（或确保环境变量正确）。
- 输出重定向（`>>`）用于追加内容。
- 使用 `tail -f` 可实时观察任务执行效果。
