## 自定义变量的必要性

在 Shell 脚本中，自定义变量用于提高代码的**可维护性**和**一致性**。例如：

- 若程序名称为 “iPhone 规谷”，在脚本中多次直接写死该名称，一旦需要更名（如改为 “谷丽商城”），需逐处修改，容易出错。
- 使用变量（如 `APP="iPhone 规谷"`）后，只需修改变量赋值一次，所有引用自动更新。
- 避免因文本替换（如 find/replace）误改其他无关内容（如注释或日志中的相同字符串）。

> **核心价值**：保证**一致性**、提升**可移植性**、简化**后期维护**。

## 自定义变量的基本语法

### 声明方式
```bash
变量名=值
```

### 语法规则
- **变量名前不能加 `$`**（仅在使用时加）。
- **等号 `=` 前后不能有空格**：
  - ❌ 错误：`A = 5` → 系统会将 `A` 视为命令，报错 “command not found”。
  - ✅ 正确：`A=5`
- 变量值若包含空格，必须用**引号包裹**（单引号 `'` 或双引号 `"`）：
  ```bash
  B="哈喽 沃的"  # 正确
  B=哈喽 沃的    # 错误：空格被视作命令分隔符
  ```

## 变量的使用与撤销

### 使用变量
通过 `$变量名` 或 `${变量名}` 引用：
```bash
A=5
echo $A        # 输出：5
echo ${A}      # 同样输出：5（推荐用于复杂上下文）
```

### 撤销变量
使用 `unset` 命令删除变量：
```bash
unset A
echo $A        # 输出为空（变量未定义）
```

> **注意**：`unset` 很少使用，因为：
> - Shell 变量占用内存极小（通常几 KB）。
> - 若需重用同名变量，建议使用**不同变量名**以避免混淆。

## 只读（静态）变量

使用 `readonly` 声明不可修改、不可删除的变量：
```bash
readonly F="M.A"
echo $F        # 输出：M.A
F="new"        # 报错：cannot modify readonly variable
unset F        # 报错：cannot unset readonly variable
```

> **适用场景**：常量配置（如版本号、API 地址等）。

## 注意事项与常见问题

### 1. 所有变量默认为字符串
Shell 不区分数据类型，即使赋值为数字，也视为字符串：
```bash
C=1+2
echo $C        # 输出：1+2（不是 3）
```
> 如需算术运算，需使用 `expr`、`$(( ))` 或 `let`。

### 2. 空格是命令分隔符
Shell 中空格具有“分段”作用，会将命令拆分为多个部分：
- `B=哈喽 沃的` → 被解析为：执行命令 `B=哈喽`，再执行命令 `沃的`。
- 必须用引号包裹含空格的值。

### 3. 引号选择
- **双引号 `"`**：允许变量展开（如 `"Hello $USER"`）。
- **单引号 `'`**：完全字面量（如 `'Hello $USER'` 输出原字符串）。

## 全局环境变量（export）

默认情况下，变量仅在当前 Shell 会话或脚本中有效。若希望子进程（如其他脚本）也能访问，需使用 `export` 导出为**环境变量**：

```bash
F="M.A"
export F       # 或写作 export F="M.A"
```

### 验证效果
- 在当前 Shell：`echo $F` → `M.A`
- 在新脚本 `hello.sh` 中：
  ```bash
  #!/bin/bash
  echo $F      # 若已 export，则输出 M.A；否则为空
  ```
- 若未 `export`，子脚本无法继承该变量。

> **类比**：`export` 类似于“发表科学论文”，让全局知晓该变量的存在。
