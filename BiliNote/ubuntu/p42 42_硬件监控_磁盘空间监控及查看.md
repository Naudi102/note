## 磁盘空间监控的重要性

在 Ubuntu（或任何 Linux）系统中，随着软件安装、系统更新和用户操作，磁盘空间会逐渐被消耗。若不及时监控，可能导致系统运行异常甚至崩溃。因此，定期检查磁盘使用情况是系统管理的重要环节。

- 安装 Ubuntu 时分配了 40GB 磁盘空间。
- 经过一段时间使用（包括系统更新等），已使用约 15GB，剩余约 23GB，目前仍较充足。
- 需要重点关注 **根目录（/）** 所在分区的使用率，因其承载系统核心文件。

## df 命令：查看整体磁盘使用情况

`df`（Disk Free）命令用于显示文件系统的磁盘空间使用情况。

### 基本用法
```bash
df
```
- 默认以 **KB（Kilobytes）** 为单位显示。
- 输出列包括：
  - **Filesystem**：文件系统设备名（如 `/dev/sda1`）
  - **Size**：总容量
  - **Used**：已使用空间
  - **Avail**：可用空间
  - **Use%**：使用百分比
  - **Mounted on**：挂载点（相当于 Windows 的 C 盘、D 盘概念）

> 在 Linux 中，所有存储设备都通过挂载点统一接入根目录 `/` 下，而非独立盘符。

### 人性化显示（推荐）
```bash
df -h
```
- `-h`（human-readable）：自动以 KB、MB、GB 等合适单位显示，便于阅读。
- 示例输出：根分区 `/` 显示为 40G 总容量，已用 15G，剩余 23G，使用率约 38%。

## du 命令：查看目录或文件的磁盘占用

`du`（Disk Usage）用于查看指定目录或文件实际占用的磁盘空间。

### 基本用法
```bash
du
```
- 默认递归列出当前目录下所有子目录和文件的大小（单位为 KB）。
- 但仅显示文件夹本身元数据（通常为 4KB），无法反映真实内容占用。

### 查看特定目录
```bash
du /path/to/dir
```
例如：
```bash
du /lib
```
- 显示 `/lib` 目录及其子项的磁盘占用。

### 人性化显示 + 递归详情
```bash
du -ah /path/to/dir
```
- `-a`：显示所有文件和目录（而不仅是目录）
- `-h`：人性化单位（如 K、M、G）

示例：
```bash
du -ah /lib
```
- 可看到具体文件如 `4.0K`、`1.6M`、`66M` 等，总计约 66MB。

### 仅显示总大小（常用）
```bash
du -sh /path/to/dir
```
- `-s`（summarize）：仅显示总占用，不展开子项。
- 结合 `-h`，可快速获取目录总大小，如 `116M`。

## 实用组合参数与技巧

| 命令 | 作用 |
|------|------|
| `df -h` | 查看所有挂载点的磁盘使用情况（推荐日常使用） |
| `du -sh /some/dir` | 快速查看某目录总占用 |
| `du -ah /some/dir` | 详细列出目录内每个文件/子目录的大小 |
| `df /` | 仅查看根分区使用情况 |

> 注意：`ls -l` 或 `ls -lh` 显示的是文件逻辑大小，而 `du` 显示的是**实际磁盘块占用**，两者可能不同（因文件系统块对齐）。

## 磁盘空间不足的风险与应对

- 当磁盘使用率 **超过 90%** 时，系统可能：
  - 无法创建临时文件
  - 日志写入失败
  - 软件安装/更新中断
  - 甚至导致系统卡死或无法启动

> 类比 Windows 中“C 盘爆红”的情况——此时必须立即清理或扩容。

### 应对措施
1. **清理无用文件**：
   - 删除旧日志（`/var/log`）
   - 清理缓存（`apt clean`、`~/.cache`）
   - 卸载不用的软件包
2. **扩展存储**：
   - 添加新硬盘并挂载到新路径（如 `/data`）
   - 使用 LVM 动态扩容（高级）

> 对于多用户系统，每个用户默认会分配少量空间（如 300KB），但现代硬盘容量下影响极小。

## AI 总结

本节教程重点介绍了 Ubuntu 系统中磁盘空间监控的两个核心命令：`df` 和 `du`。`df -h` 用于全局查看各分区使用情况，而 `du -sh` 则用于精准定位大目录或文件。当磁盘使用率接近上限（如 >90%）时，系统稳定性将严重受损，需及时清理或扩容。掌握这些命令，是 Linux 系统日常维护和故障排查的基础技能。
