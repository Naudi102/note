## 工具概述

`cut` 是 Linux/Unix 系统中用于从文本文件或标准输入中“剪切”出指定部分数据的命令行工具。  
主要用途：**从结构化文本中提取特定列或字符**，常用于日志处理、配置解析、数据清洗等场景。

> **注意**：视频中口误将 `cut` 称为 “卡车”、“Cars”、“卡层”、“检妻” 等，实际命令为 `cut`。

---

## `cut` 基本语法与选项

```bash
cut [选项] 文件名
```

常用选项：
- `-f, --fields=LIST`：按**字段（列）** 提取，需配合分隔符使用。
- `-d, --delimiter=DELIM`：指定字段分隔符（默认为制表符 `\t`）。
- `-c, --characters=LIST`：按**字符位置**提取（不依赖分隔符）。

---

## 按字段（列）提取：`-f` 选项

### 工作原理
- 将每一行按指定分隔符（如空格、制表符、冒号等）**分割成多个字段（列）**。
- 使用 `-f N` 提取第 N 列。

### 默认分隔符
- 默认使用 **制表符（Tab）**。
- 若数据以**空格**分隔，需显式指定 `-d ' '`。

### 示例
假设文件 `data.txt` 内容如下（以空格分隔）：
```
东玩 我来了 非常开心
深圳 我一来一 乐意
```

提取第一列：
```bash
cut -d ' ' -f1 data.txt
# 输出：
# 东玩
# 深圳
```

提取第二列：
```bash
cut -d ' ' -f2 data.txt
# 输出：
# 我来了
# 我一来一
```

## 按字符位置提取：`-c` 选项

### 特点
- 不依赖分隔符，直接按**字符在行中的位置**提取。
- 即使该位置是空格，也会被提取。

### 示例
对同一文件 `data.txt`：

提取第1个字符：
```bash
cut -c1 data.txt
# 输出：
# 东
# 深
```

提取第3个字符：
```bash
cut -c3 data.txt
# 输出（可能包含空格）：
# （空格）
# （空格）
```

> 注意：中文字符在某些系统中可能占多个字节，但 `cut -c` 按字节处理，可能导致乱码。建议在纯 ASCII 环境下使用 `-c`。

## 结合 `grep` 实现行列联合筛选

`cut` **只能按列筛选**，无法直接按行过滤。  
实际开发中通常结合 `grep`（行过滤） + `cut`（列提取）使用，通过**管道（`|`）** 连接。

典型流程：
```bash
命令 | grep "关键词" | cut -d'分隔符' -fN
```

## 实战案例

### 案例一：提取 `$PATH` 环境变量中第3段路径

`$PATH` 以冒号 `:` 分隔，例如：
```
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin
```

目标：获取**第2个冒号之后的所有路径**（即从第3段开始的内容）。

操作步骤：
```bash
echo $PATH | cut -d':' -f3-
```

说明：
- `-f3-` 表示从第3字段到末尾。
- 若只写 `-f3`，则仅输出第3段。

### 案例二：提取网卡 IP 地址

使用 `ip addr`（视频中误说为 `AV 缝费的`，应为 `ip a` 或 `ip addr show`）查看网络接口信息。

目标：从 `ens33` 网卡信息中提取 IPv4 地址（如 `192.168.6.101`）。

#### 方法一：按空格分隔 + 字段定位
```bash
ip addr show ens33 | grep "inet " | cut -d' ' -f6 | cut -d'/' -f1
```

步骤解析：
1. `ip addr show ens33`：显示指定网卡信息。
2. `grep "inet "`：过滤出包含 IPv4 地址的行（注意空格避免匹配 `inet6`）。
3. `cut -d' ' -f6`：按空格分隔，取第6字段（通常是 `IP/掩码` 格式）。
4. `cut -d'/' -f1`：再按 `/` 分割，取 IP 部分。

#### 方法二：利用 `inet` 关键字作为分隔符（更健壮）
```bash
ip addr show ens33 | grep "inet " | cut -d'i' -f2 | cut -d' ' -f2
```

> 视频中提到可使用 `-d 'i'` 将 `inet 192.168.6.101/24` 分割为两部分，再提取 IP。但此方法依赖文本结构，不如方法一通用。

## AI 总结

`cut` 是 Linux 中高效提取结构化文本字段的核心工具，支持按字段（`-f`）或字符（`-c`）进行剪切。实际应用中，常与 `grep`、`echo`、`ip` 等命令通过管道组合，实现“先行过滤、再列提取”的数据处理流程。掌握 `-d` 指定分隔符和 `-fN-` 提取范围的用法，能显著提升 Shell 脚本的数据处理能力。注意中文环境下 `-c` 可能因编码问题导致异常，建议优先使用 `-f` 处理字段数据。
