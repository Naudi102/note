## 修改用户密码

- 使用 `passwd` 命令修改当前用户的密码：
  ```bash
  passwd
  ```
- 若需修改**其他用户**的密码，必须具备更高权限（如 root 或使用 `sudo`）：
  ```bash
  sudo passwd username
  ```
  - 示例：`sudo passwd tangsen`
- 即使设置弱密码（如 `123456`），只要使用 `sudo` 提权，系统仍会接受（尽管会提示“无效密码”警告）。
- **注意**：命令名为 `passwd` 而非 `pwd`，因为 `pwd` 已被用于 “Print Working Directory”。

## 查看用户信息

- 使用 `id` 命令查看指定用户的基本信息：
  ```bash
  id username
  ```
  - 输出包含 UID（用户 ID）、GID（组 ID）及所属组列表。
  - 示例：`id atguigu` → UID 为 1000。

- **UID 分配规则**：
  - UID 0：root 用户。
  - UID 1–999：系统保留用户（如 daemon、syslog 等）。
  - UID ≥1000：普通用户（第一个创建的普通用户通常为 1000）。

- 查看所有用户信息可直接读取 `/etc/passwd` 文件：
  ```bash
  cat /etc/passwd
  ```
  - 每行格式：`用户名:密码占位符:UID:GID:描述:家目录:默认Shell`
  - 示例条目：
    ```
    root:x:0:0:root:/root:/bin/bash
    atguigu:x:1000:1000::/home/atguigu:/bin/bash
    tangsen:x:1001:1001::/home/tangsen:/bin/bash
    ```

## 切换用户

- 使用 `su`（Switch User）命令切换到其他用户：
  ```bash
  su username
  ```
  - 示例：`su tangsen` → 输入该用户的密码后即可切换。
- **Ubuntu 默认不启用 root 密码**，因此**不能直接 `su root`**。
  - 这是 Ubuntu 的安全设计，与 CentOS 等发行版不同。
  - 需要 root 权限时应使用 `sudo`，而非切换到 root 用户。
- 切换成功后，命令行提示符和工作目录会变为目标用户的家目录。

## 用户家目录与 PATH 环境变量

- 每个用户拥有独立的**家目录**（如 `/home/atguigu`、`/home/tangsen`）。
  - 不同用户无法直接访问彼此的家目录（受权限限制）。
- 用户的 `PATH` 环境变量可能不同：
  - `PATH` 决定了 shell 在哪些目录中查找可执行命令。
  - 可通过 `echo $PATH` 查看当前用户的 PATH。
  - 不同用户的 PATH 可能包含其家目录下的 `~/bin` 或自定义路径。
- **影响**：同一命令名在不同用户下可能指向不同程序（若各自在 PATH 中添加了私有二进制文件）。

> 💡 类比 Windows：Windows 的“环境变量 > Path”作用相同，决定命令行可执行程序的搜索路径。

## 企业开发中的用户使用实践

- 在团队开发中，若每个开发者使用独立用户，可能导致：
  - 自定义工具或脚本仅对特定用户可用。
  - 其他成员无法使用这些命令（因 PATH 不同或权限不足）。
- **常见解决方案**：
  - 多人共用一个低权限开发账户（如 `devuser`），便于共享工具链和环境。
  - 虽不符合“最小权限原则”，但在实际协作中更高效。
- 教程指出：这种做法在企业中“很常见且正常”。

## 退出当前用户会话

- 使用 `exit` 命令退出当前 `su` 切换的用户会话：
  ```bash
  exit
  ```
  - 执行后返回到原登录用户（如从 `tangsen` 切回 `atguigu`）。
- 若多次嵌套切换，需多次 `exit` 才能完全退出。
