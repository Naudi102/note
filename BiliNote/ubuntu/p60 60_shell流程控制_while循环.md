## while 循环基本概念

- `while` 是 Shell 脚本中第二种常用的循环结构（继 `for` 循环之后）。
- 其基本逻辑是：**先判断条件，若为真则执行循环体；重复此过程，直到条件为假为止**。
- 与 `for` 循环不同，`while` **不会自动递增变量**，必须在循环体内手动更新控制变量。

## while 与 for 循环的对比

- 功能上，`while` 和 `for` 在大多数场景下可以互相替代。
- 关键区别：
  - `for` 循环通常自带变量初始化、条件判断和自增操作（如 `for i in {1..100}`）。
  - `while` 循环需要**显式声明初始变量**，并在循环体内**手动更新该变量**，否则可能陷入死循环。

## while 实现无限循环

- 可通过设置恒真条件实现无限循环，例如：
  ```bash
  while true
  do
      # 循环体
  done
  ```
  或
  ```bash
  while [ 1 -eq 1 ]
  do
      # 循环体
  done
  ```
- 这种写法常用于服务程序或监听脚本中，但需配合 `break` 或外部信号终止。
- **警告**：若无退出机制，会导致 CPU 空转，**严重消耗系统资源**，不推荐在普通脚本中使用。

## 实战示例：使用 while 计算 1 到 100 的和

### 脚本编写步骤：

1. **声明 Shebang**：
   ```bash
   #!/bin/bash
   ```

2. **初始化变量**：
   ```bash
   i=1
   sum=0
   ```

3. **编写 while 循环**：
   ```bash
   while [ $i -le 100 ]
   do
       sum=$(( $sum + $i ))
       i=$(( $i + 1 ))
   done
   ```

4. **输出结果**：
   ```bash
   echo $sum
   ```

### 完整脚本示例：
```bash
#!/bin/bash
i=1
sum=0
while [ $i -le 100 ]
do
    sum=$(( $sum + $i ))
    i=$(( $i + 1 ))
done
echo $sum
```

### 执行流程：
- 初始 `i=1`, `sum=0`
- 每次循环将 `i` 加到 `sum` 中，并将 `i` 自增 1
- 当 `i` 变为 101 时，条件 `$i -le 100` 不再成立，循环结束
- 最终输出 `5050`

- 将脚本保存为 `while.sh`
- 赋予执行权限：
  ```bash
  chmod +x while.sh
  ```
- 运行脚本：
  ```bash
  ./while.sh
  ```
- 输出结果为 `5050`，验证正确性。

## 注意事项与最佳实践

- **必须手动更新循环变量**：否则会陷入死循环。
- **避免无意义的空转无限循环**：如 `while true` 且无 `sleep` 或 `break`，会导致 CPU 占用率飙升。
- 在实际应用中（如嵌入式或服务程序），若需永久循环，应加入：
  - `sleep` 延迟以降低 CPU 负载
  - 信号处理或条件判断以安全退出
- 使用 `[ ]` 进行条件判断时，注意：
  - 运算符两侧需有空格（如 `-le`）
  - 变量建议用 `$` 引用（如 `$i`）
