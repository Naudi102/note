## 特殊变量 `$*` 与 `$@`

在 Shell 脚本中，`$*` 和 `$@` 都用于表示传递给脚本或函数的所有位置参数（即命令行参数）。

### 共同点
- 两者在大多数情况下功能非常相似。
- 默认行为都是将所有参数作为一个整体输出。
- 示例：执行 `./script.sh a b c` 后，`$*` 和 `$@` 都会输出 `a b c`。

### 关键区别（在循环中体现）
- **`$*`**：将所有参数视为**一个字符串**。在循环中，即使有多个参数，也会被当作**单次迭代**处理。
- **`$@`**：会**智能解析**每个参数，即使参数中包含空格（如 `"hello world"`），也能正确地按原始传入的参数个数进行拆分和遍历。

> 注：讲师提到，两者的具体差异将在后续“循环”章节中详细讲解，此处可先视作功能相近。

### 演示操作
1. 在脚本中添加以下内容：
   ```bash
   echo "Using \$*: $*"
   echo "Using \$@: $@"
   ```
2. 保存并执行脚本，传入多个参数：
   ```bash
   bash script.sh 一 二 三
   ```
3. 输出结果：
   ```
   Using $*: 一 二 三
   Using $@: 一 二 三
   ```

## 特殊变量 `$?`

`$?` 是一个非常重要的特殊变量，用于获取**上一条命令的退出状态码（exit status）**。

### 含义
- **`0`**：表示上一条命令**执行成功**。
- **非零值（如 1, 2, 127 等）**：表示**执行失败**，不同数值代表不同类型的错误。

### 使用场景
- 调试脚本时判断命令是否成功。
- 在条件判断中控制程序流程。

### 示例
```bash
ls /existing_dir
echo $?  # 输出 0（成功）

ls /nonexistent_dir
echo $?  # 输出非零值（如 2，表示“没有该文件或目录”）
```

> 注意：只要上一条命令正常结束，`$?` 就为 0；一旦出错，其值即为对应的错误码，具体数值取决于错误类型。

## AI 总结

本节介绍了 Shell 脚本中的三个关键特殊变量：`$*`、`$@` 和 `$?`。其中，`$*` 与 `$@` 均用于获取所有命令行参数，在简单输出时表现一致，但在循环处理带空格的参数时行为不同——`$@` 更安全、更精确。而 `$?` 则用于捕获上一条命令的执行状态，是脚本调试和错误处理的核心工具。掌握这三个变量，对编写健壮的 Shell 脚本至关重要。
