## Shell 中的变量赋值与运算问题

在 Shell 脚本中，直接将表达式（如 `1+2`）赋值给变量时，**Shell 不会自动进行数学计算**，而是将其视为**字符串（文本）**处理。

例如：
```bash
A=1+2
echo $A  # 输出结果为 "1+2"，而非 3
```

这是因为 Shell 默认将等号右侧的内容当作纯文本存储，**不具备自动识别并计算算术表达式的能力**。

## Shell 运算的两种语法形式

为了让 Shell 执行算术运算，必须使用特定的**运算语法结构**。视频中介绍了两种常用方式：

1. **双小括号形式**：`$(( 表达式 ))`
2. **单中方括号形式**：`$[ 表达式 ]`（注：此处讲师口误称“中括号”，实际指 `$[ ]` 语法）

> ⚠️ 注意：严格来说，`$[ ]` 是 Bash 的旧式算术扩展语法，而 `$(( ))` 是 POSIX 标准推荐的现代写法。但视频中将 `$[ ]` 称为“一个中括号”，按其表述记录。

## 推荐使用 $[ ] 还是 $(( ))？

讲师**推荐使用 `$[ ]` 形式**，理由如下：

- `$(( ))` 中的内层小括号在数学中本就表示“优先运算”（如 `(8+2)`），若在外层再套一层 `$(( ))`，视觉上容易混淆，不够简洁。
- `$[ ]` 结构更直观、简洁，不易看错。

> 💡 实际建议：虽然视频推荐 `$[ ]`，但在现代 Shell 编程中，**`$(( ))` 是标准且广泛支持的写法**，而 `$[ ]` 已被 Bash 标记为过时（deprecated）。建议学习者优先掌握 `$(( ))`。

## 实际示例演示

### 示例 1：使用 `$[ ]` 进行加法运算
```bash
B=$[1+2]
echo $B  # 输出 3
```

### 示例 2：使用 `$(( ))` 进行加法运算
```bash
B=$((1+2+8))
echo $B  # 输出 11
```

两种语法效果相同，均能正确执行算术计算。

## 关键结论

- **Shell 不会自动计算变量赋值中的表达式**，必须显式使用算术扩展语法。
- 可用语法：
  - `$(( expression ))`（标准推荐）
  - `$[ expression ]`（旧式，部分 Shell 支持）
- 虽然视频推荐 `$[ ]`，但从兼容性和规范性角度，**应优先使用 `$(( ))`**。
- 加减乘除等基本运算均可在这些结构中正常执行。
