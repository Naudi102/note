## 核心命令：`netstat`

- `netstat` 是 Linux（包括 Ubuntu）中用于监控网络连接、路由表、接口统计、伪装连接、多播成员等信息的命令。
- 默认执行 `netstat` 会输出大量内容，包括本地套接字（socket）和网络连接。
- **真正用于外部网络通信的连接**通常位于输出的上半部分，需重点关注。

## 常用参数组合

为高效查看端口占用情况，推荐使用以下参数组合：

- **`-p`（或 `--programs`）**：
  - 显示每个连接对应的**进程 ID（PID）和进程名**。
  - 若不加 `-p`，只能看到端口被谁连接，但无法知道是哪个进程在使用，不利于后续管理。
  - 普通用户可能因权限不足而无法读取完整信息，此时需配合 `sudo` 使用。

- **`-n`（或 `--numeric`）**：
  - 以**数字形式**显示地址和端口号，而非解析为服务名称（如 `ssh` → `22`）。
  - 有助于快速识别实际使用的端口号，避免混淆。

- **`-l`（或 `--listening`）**：
  - 仅显示**监听状态**的套接字（即服务正在等待连接的端口）。

- **常用组合**：
  - `netstat -tulnp`：查看所有 TCP/UDP 监听端口及其对应进程（最实用）。
  - `netstat -anp`：显示所有连接（包括非监听）的详细信息。
  - `netstat -nl`：仅显示监听端口的数字形式（不含进程信息）。

> 示例：  
> ```bash
> sudo netstat -tulnp | grep 6010
> ```
> 可快速定位 6010 端口是否被占用及对应 PID。

## 查看与释放端口的完整流程

1. **查看端口占用**：
   ```bash
   sudo netstat -tulnp | grep <端口号>
   ```
   例如查找 6010 端口：
   ```bash
   sudo netstat -tulnp | grep 6010
   ```

2. **识别占用进程**：
   - 输出中会显示协议、本地地址、外部地址、状态、PID/程序名。
   - 示例输出：
     ```
     tcp6 0 0 :::6010 :::* LISTEN 24938/sshd
     ```

3. **终止占用进程（谨慎操作）**：
   ```bash
   sudo kill <PID>
   # 或强制终止
   sudo kill -9 <PID>
   ```
   - 如 `sudo kill 24938` 可释放 6010 端口。
   - 终止后再次运行 `netstat` 命令，确认端口已释放。

（展示 kill 进程后，再次运行 netstat 确认 6010 端口不再被占用）

## 操作注意事项

- **权限问题**：普通用户执行 `netstat -p` 时可能提示“非 root 无法读取信息”，必须使用 `sudo`。
- **进程重要性判断**：
  - 若占用端口的是 **SSH 连接**（如 `sshd`），且为非关键会话，可安全终止。
  - 若是 **数据库（如 MySQL、PostgreSQL）**、**Web 服务（如 Nginx、Apache）** 或其他关键服务，**切勿随意 kill**，需与团队协商。
- **端口归属识别**：
  - 本地 IP（如 `192.168.6.1`）通常代表远程客户端（如 Windows 主机）通过 SSH 连接到 Ubuntu。
  - `:::` 表示 IPv6 监听所有接口，`0.0.0.0` 表示 IPv4 监听所有接口。

## AI 总结

本节教程详细介绍了在 Ubuntu 系统中使用 `netstat` 命令监控网络端口占用的方法。核心在于结合 `-p`（显示进程）、`-n`（数字端口）、`-l`（监听端口）等参数，精准定位占用端口的进程。通过 `sudo netstat -tulnp | grep <端口>` 可快速查询，再根据进程重要性决定是否使用 `kill` 命令释放端口。操作时务必注意权限和进程关键性，避免误杀重要服务。该技能对嵌入式开发、服务器运维和网络调试具有重要实用价值。
