## 实战目标
本次实战任务包含以下三个步骤：
1. 下载最新版 **Apache Flume** 框架的 **二进制（Binary）压缩包**（非源码包）。
2. 将压缩包上传至 `/opt/module/` 目录，并解压。
3. 在解压后的目录中，**查找所有超过1MB的 `.jar` 文件**。

> ⚠️ 注意：必须下载 **Binary**（可直接运行）版本，而非 **Source**（需编译）版本。

## 第一步：下载Flume框架安装包

### 官方渠道
- Apache Flume 是 Apache 基金会管理的开源项目。
- 官网地址：`https://flume.apache.org/`
- 在官网下载页面中，通常提供两类压缩包：
  - **Binary**（`.tar.gz` 或 `.zip`）：已编译好的可执行版本，**推荐下载**。
  - **Source**（`.tar.gz`）：源代码包，需自行编译，不适用于本场景。

### 推荐镜像源（加速下载）
由于官方服务器可能较慢，建议使用国内镜像站，例如：
- **清华大学开源软件镜像站**  
  网址：`https://mirrors.tuna.tsinghua.edu.cn/`
  - 进入后搜索 “apache”
  - 进入 `apache/flume/` 目录
  - 选择最新版本（如 `1.9.0/`）
  - 下载 `apache-flume-<version>-bin.tar.gz`

> ✅ 示例文件名：`apache-flume-1.9.0-bin.tar.gz`  
> ❌ 避免下载：`apache-flume-1.9.0-src.tar.gz`

## 第二步：上传并解压安装包

### 1. 创建目标目录并设置权限
```bash
# 确保 /opt/module/ 目录存在且当前用户有写权限
sudo mkdir -p /opt/module/
sudo chown $USER:$USER /opt/module/
```

### 2. 上传压缩包
- 将下载的 `apache-flume-*-bin.tar.gz` 上传至 `/opt/module/` 目录（可通过 SFTP、scp 或图形界面操作）。

### 3. 解压命令
进入 `/opt/module/` 目录，执行解压命令：
```bash
cd /opt/module/
tar -zxvf apache-flume-1.9.0-bin.tar.gz
```

> 🔍 参数说明：
> - `z`：使用 gzip 解压
> - `x`：解压
> - `v`：显示过程
> - `f`：指定文件名

解压后将生成类似 `apache-flume-1.9.0-bin/` 的目录。

## 第三步：查找超过1MB的 `.jar` 文件

### 方法一：使用 `find` 命令（推荐）
```bash
find ./apache-flume-1.9.0-bin -name "*.jar" -size +1M
```

> 🔍 参数说明：
> - `-name "*.jar"`：匹配以 `.jar` 结尾的文件
> - `-size +1M`：文件大小 **大于 1MB**（注意：`M` 必须大写）

### 方法二：使用 `find` + `grep` 管道组合
适用于不熟悉通配符或正则表达式的用户：
```bash
find ./apache-flume-1.9.0-bin -size +1M | grep "\.jar$"
```

> 🔍 说明：
> - 先找出所有 >1MB 的文件
> - 通过管道 `|` 传递给 `grep`
> - `grep "\.jar$"` 筛选出以 `.jar` 结尾的行

### 输出示例
```text
./apache-flume-1.9.0-bin/lib/flume-ng-node-1.9.0.jar
./apache-flume-1.9.0-bin/lib/hadoop-common-2.7.3.jar
...
```

> 💡 提示：`.jar` 文件通常位于 `lib/` 目录下，但使用 `find` 可确保全面扫描。

## AI 总结

本节实战演练完整演示了在 Ubuntu 系统中处理开源框架的标准流程：**从官方或镜像站下载 Binary 版本压缩包 → 上传至指定目录 → 使用 `tar -zxvf` 解压 → 利用 `find` 命令结合文件名和大小条件精准定位目标文件**。重点强调了 Binary 与 Source 包的区别，并提供了两种查找 `.jar` 文件的方法，兼顾了不同用户对命令行熟练度的需求。该流程具有通用性，适用于 Hadoop、Spark、Kafka 等大多数 Apache 开源项目。
