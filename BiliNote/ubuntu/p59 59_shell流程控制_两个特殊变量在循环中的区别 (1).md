## 核心概念：`$*` 与 `$@` 的区别

在 Shell 脚本中，`$*` 和 `$@` 都用于表示传递给脚本或函数的所有位置参数（即命令行参数）。  
在**多数情况下**（尤其是未加双引号时），二者表现一致；但在**加双引号**的上下文中，它们的行为存在关键差异：

- **`$*`**：将所有参数视为**一个整体字符串**。
- **`$@`**：将每个参数**分别保留为独立的字段**，即使被双引号包围。

这一区别在 `for` 循环中尤为明显，直接影响循环次数和参数处理方式。

## 无双引号情况下的行为

当在 `for` 循环中使用 **不带双引号** 的 `$*` 或 `$@` 时：

```bash
for i in $*; do
  echo $i
done

for i in $@; do
  echo $i
done
```

- 两者行为**完全相同**。
- Shell 会根据 **IFS（内部字段分隔符）** 对参数进行单词分割。
- 每个独立的词（word）会被当作一次循环项。

> 示例：执行脚本 `./script.sh "蔡老师" "徐老师" "昆老师"`  
> 输出：三次循环，分别打印“蔡老师”、“徐老师”、“昆老师”。

## 带双引号情况下的行为

当在 `for` 循环中使用 **带双引号** 的形式时，行为出现显著差异：

### 使用 `"$*"`

```bash
for i in "$*"; do
  echo $i
done
```

- 所有参数被**合并为一个字符串**，以空格连接。
- 整个字符串被视为**单个参数**。
- **循环仅执行一次**。

> 示例：`./script.sh "蔡老师" "徐老师" "昆老师"`  
> 输出：`蔡老师 徐老师 昆老师`（一行）

### 使用 `"$@"`

```bash
for i in "$@"; do
  echo $i
done
```

- 每个原始参数**保持独立**，即使包含空格也不会被拆分。
- **循环次数 = 参数个数**。
- 这是**推荐做法**，尤其在处理含空格的文件名或用户输入时。

> 示例：同上输入  
> 输出：三行，分别为“蔡老师”、“徐老师”、“昆老师”。

## 实践建议

- **始终优先使用 `"$@"`** 来遍历外部传入的参数。
  - 它能正确保留每个参数的边界，避免因空格或特殊字符导致的错误。
- 仅在需要将所有参数拼接成一个字符串时，才考虑使用 `"$*"`。
- 在编写健壮的 Shell 脚本时，**务必对 `$@` 加双引号**：`"$@"`。

> 教程中强调：“往后如果大家想要去表示外界传进来的每一个参数的话，你就应该用 `$@`。”

## AI 总结

在 Shell 脚本中，`$*` 和 `$@` 在无引号时行为一致，但在双引号包裹下表现截然不同：`"$*"` 将所有参数合并为单一字符串，而 `"$@"` 保留每个参数的独立性。在 `for` 循环中处理用户输入或命令行参数时，应始终使用 `"$@"` 以确保参数完整性与脚本健壮性。这是编写可靠 Shell 脚本的关键实践之一。
