## 流程控制概述

在编程中，**流程控制（Flow Control）** 是指控制程序执行顺序的机制。早期的程序常使用“跳转”（如 goto）等混乱方式，导致代码难以理解和维护。

现代编程理论将流程控制归纳为 **三种基本结构**，即可实现所有逻辑需求：

1. **顺序结构（自然顺序）**  
   - 默认从上到下逐行执行。
   - 无需特殊语法，是程序的基础执行方式。

2. **条件判断（分支结构）**  
   - 根据条件决定执行哪段代码（如分岔路口）。
   - 例如：年龄 ≥ 18 允许进入，否则拒绝。

3. **循环结构**  
   - 重复执行某段代码，直到满足特定条件。
   - 类似“练功直到达标”，达标即退出循环。

> 💡 实际学习中，只需重点掌握 **条件判断** 和 **循环** 两种结构。

## 条件判断（if 语句）

Shell 中最常用的条件判断结构是 `if` 语句，其核心逻辑为：
- **如果条件成立，则执行某操作；否则执行另一操作（可选）**。

### 单分支 if 语法

基本格式如下（两种写法）：

```bash
# 写法一：单行（使用分号）
if [ condition ]; then
    commands
fi

# 写法二：多行（换行代替分号）
if [ condition ]
then
    commands
fi
```

关键点：
- 条件表达式必须用 **方括号 `[ ]`** 包裹，且 **前后需有空格**。
- `then` 可在同一行（加分号）或下一行。
- `fi` 是 `if` 的反写，表示 `if` 语句结束，**必须配对使用**。

### 多分支 if-elif-else 语法

支持多个条件分支：

```bash
if [ condition1 ]; then
    commands1
elif [ condition2 ]; then
    commands2
else
    commands3
fi
```

注意事项：
- `elif` 表示“否则如果”，隐含 **前面所有条件都不成立**。
- 例如：若第一个条件是 `$1 -eq 18`，则 `elif` 分支中无需再判断 `$1 -le 18`，因为已隐含此逻辑。
- 避免条件重复，保持逻辑清晰。

## 实战示例：if.sh 脚本编写

目标：编写脚本 `if.sh`，根据输入参数输出不同内容：
- 输入 `1` → 输出 “班长真帅”
- 输入 `2` → 输出 “蔡老师真美”
- 其他输入 → 无输出

### 脚本代码

```bash
#!/bin/bash

if [ "$1" -eq 1 ]; then
    echo "班长真帅"
elif [ "$1" -eq 2 ]; then
    echo "蔡老师真美"
else
    :
fi
```

说明：
- `"$1"` 表示脚本的第一个参数，**建议加双引号防止空值错误**。
- `-eq` 用于整数相等比较（注意：字符串比较用 `=` 或 `==`）。
- `:` 是 Shell 中的空命令（no-op），表示“什么都不做”。

```bash
./if.sh 2   # 输出：蔡老师真美
./if.sh 3   # 无输出
```

## 常见错误与调试

1. **语法错误：缺少 `then`**
   - 错误示例：
     ```bash
     if [ "$1" -eq 1 ]
         echo "班长真帅"
     fi
     ```
   - 报错：`syntax error near unexpected token 'fi'`
   - 修复：必须包含 `then`（即使换行也需显式写出）。

2. **方括号空格缺失**
   - 错误：`if [$1 -eq 1]` → 正确：`if [ "$1" -eq 1 ]`

3. **未处理非数字输入**
   - 若传入非整数（如 `abc`），`-eq` 会报错。
   - 建议：先验证输入是否为数字（本教程未涉及，但实际开发需考虑）。
