## Shell 条件判断基础

在 Shell 脚本中，条件判断用于检测某个表达式是否为真（true）或假（false）。其返回值通过特殊变量 `$?` 获取：

- 返回 `0` 表示 **条件为真（true）**
- 返回非 `0`（通常为 `1`）表示 **条件为假（false）**

有两种主要语法形式进行条件判断：

1. **`test` 命令**  
   ```bash
   test 条件
   ```

2. **`[ ]` 结构（更常用）**  
   ```bash
   [ 条件 ]
   ```
   - 注意：`[` 和 `]` 两边必须有空格
   - 这是 `test` 命令的等价写法，但更简洁、可读性更强，尤其在 `if` 语句中广泛使用

## 数值比较操作符

在 Shell 中，**不能使用 `<`、`>` 等符号直接比较数字**，而应使用以下专用操作符（均需在 `test` 或 `[ ]` 中使用）：

| 操作符 | 含义         | 示例                     |
|--------|--------------|--------------------------|
| `-eq`  | 等于（equal）| `[ 2 -eq 2 ]` → true     |
| `-ne`  | 不等于（not equal）| `[ 2 -ne 3 ]` → true |
| `-lt`  | 小于（less than）| `[ 2 -lt 3 ]` → true   |
| `-le`  | 小于等于（less or equal）| `[ 2 -le 2 ]` → true |
| `-gt`  | 大于（greater than）| `[ 3 -gt 2 ]` → true  |
| `-ge`  | 大于等于（greater or equal）| `[ 3 -ge 3 ]` → true |

> 示例：
> ```bash
> [ 2 -ge 3 ]; echo $?  # 输出 1（false）
> [ 2 -ge 2 ]; echo $?  # 输出 0（true）
> ```

## 文件与权限判断

Shell 提供多种文件属性测试操作符，常用于脚本中验证文件状态：

### 文件存在性与类型
| 操作符 | 含义                     | 示例                     |
|--------|--------------------------|--------------------------|
| `-e`   | 文件/目录是否存在        | `[ -e /etc/passwd ]`     |
| `-f`   | 是否为普通文件           | `[ -f script.sh ]`       |
| `-d`   | 是否为目录               | `[ -d /home ]`           |

### 权限判断
| 操作符 | 含义         | 示例                   |
|--------|--------------|------------------------|
| `-r`   | 是否可读     | `[ -r file.txt ]`      |
| `-w`   | 是否可写     | `[ -w file.txt ]`      |
| `-x`   | 是否可执行   | `[ -x script.sh ]`     |

> 示例演示（以当前用户对 `/etc/passwd` 的权限为例）：
> ```bash
> [ -r /etc/passwd ]; echo $?  # 0（可读）
> [ -w /etc/passwd ]; echo $?  # 0（部分系统可写）
> [ -x /etc/passwd ]; echo $?  # 1（不可执行）
> ```

## 字符串与空值判断

`[ ]` 结构本身也可用于判断字符串是否为空：

- `[ -z "$str" ]`：判断字符串是否为空（zero length）
- `[ -n "$str" ]`：判断字符串是否非空（non-zero length）
- 单独使用 `[ "$str" ]`：若 `$str` 非空则为真

> 特殊情况：
> ```bash
> [ ]; echo $?    # 1（false，空条件）
> [ "hello" ]; echo $?  # 0（true）
> ```

> 注：该用法较少见，通常显式使用 `-z` 或 `-n` 更清晰。

## 多条件组合：逻辑与（&&）和逻辑或（||）

Shell 支持通过逻辑操作符组合多个命令或条件：

### 逻辑与（`&&`）
- 仅当**前一个命令成功（返回 0）**，才执行后一个命令
- 类似“短路与”

### 逻辑或（`||`）
- 仅当**前一个命令失败（返回非 0）**，才执行后一个命令
- 类似“短路或”

> 示例：
> ```bash
> # 成功 && 执行
> echo "hello" && echo "OK"        # 输出两行
>
> # 失败 && 不执行
> false && echo "OK"               # 无输出
>
> # 失败 || 执行
> false || echo "Fallback"         # 输出 Fallback
> ```

> 复合示例：
> ```bash
> [ -f nonexist.txt ] && echo "Found" || echo "Not found"
> # 若文件不存在，输出 "Not found"
> ```

> 注意：命令的最终退出状态由**最后一个执行的命令**决定，可通过 `$?` 查看。
