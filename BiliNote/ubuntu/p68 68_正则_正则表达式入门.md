## 正则表达式概述

- 正则表达式（Regular Expression，简称 **Regex**）是一种用于字符串匹配的抽象模式语言。
- 应用广泛，但完全掌握难度高，如同“背字典”；然而，**完全不会也不可行**。
- 本课程聚焦于 **Shell 环境下的正则表达式**（如 `grep` 命令中使用），不同编程语言（如 Python、JavaScript）的正则语法存在差异，需查阅对应文档。
- 目标：掌握**常用基础语法**，能理解并查询复杂表达式。

## 基础匹配方式

- **常规匹配（Literal Matching）**：
  - 直接匹配字面内容。例如 `grep "A"` 会匹配所有包含字母 `A` 的行。
  - 之前课程中已介绍过 `grep` 的基本用法。

- **正则匹配（Pattern Matching）**：
  - 使用特殊符号构建**抽象模式**，匹配符合规则的字符串，而非固定文本。

## 常用特殊字符

### 行首与行尾锚点（^ 和 $）

- **`^`（caret）**：表示**一行的开始**。
  - 示例：`grep "^A"` 仅匹配以 `A` 开头的行。
  - *![](http://localhost:8000/static/screenshots/screenshot_016a8fa87-2fd2-437f-aa50-797146c7626e.jpg)*

- **`$`（dollar）**：表示**一行的结束**。
  - 示例：`grep "N$"` 仅匹配以 `N` 结尾的行。
  - *![](http://localhost:8000/static/screenshots/screenshot_18716daf7-5fce-4f58-bd69-113476a608f3.jpg)*

- **`^$`**：匹配**空行**（行开始后立即结束）。
  - 需在文件中手动添加空行才能演示效果。
  - *![](http://localhost:8000/static/screenshots/screenshot_2f652d7f2-9719-4abf-b97c-39142c05bae0.jpg)*

### 通配符（.）

- **`.`（点）**：匹配**任意单个字符**（除换行符外）。
  - 示例：`grep "R..T"` 匹配以 `R` 开头、`T` 结尾、中间任意两个字符的字符串（如 `ROOT`, `R12T`）。
  - 可用于模糊记忆单词结构时的搜索。

### 星号（*）

- **`*`（星号）**：表示**前一个字符出现任意次（包括 0 次）**。
  - 示例：`grep "RO*T"` 匹配 `RT`（O 出现 0 次）、`ROT`（1 次）、`ROOT`（2 次）等。
  - 注意：`*` 本身不匹配任意字符，而是修饰前一个字符的重复次数。

### 组合用法：.*（任意内容）

- **`.*`**：`.`（任意字符） + `*`（任意次） = **匹配任意长度的任意内容（包括空）**。
  - 常用于已知开头和结尾、中间内容未知的情况。
  - 示例：`grep "^nm.*in$"` 匹配以 `nm` 开头、`in` 结尾的行，中间可为任意内容。

### 字符集合（[ ]）

- **`[...]`（方括号）**：定义**字符集合**，匹配其中任意一个字符。
  - 枚举：`[68]` 匹配 `6` 或 `8`。
  - 范围：`[0-9]` 匹配任意数字；`[a-z]` 匹配任意小写字母；`[A-F]` 匹配 A 到 F。
  - 与 `*` 结合：`[0-9]*` 匹配任意长度的数字串（包括空字符串）。
    - 若需确保至少一个数字，应写为 `[0-9][0-9]*` 或使用 `+`（但 Shell 基础正则中 `+` 需转义）。
  - 示例：`grep "R[ABC]*T"` 匹配 `RT`, `RAT`, `RABT`, `RCCT` 等。

### 转义字符（\）

- **`\`（反斜杠）**：用于**转义特殊字符**，使其失去特殊含义，作为普通字符匹配。
  - 示例：要匹配字面 `$` 符号（如字符串 `A$B`），需写为 `grep "A\$B"`。
  - 若不加 `\`，`$` 会被解释为行尾锚点，导致匹配失败。
  - *![](http://localhost:8000/static/screenshots/screenshot_3b4d1e6de-d7be-4da0-9137-dcf243b84e29.jpg)*

## 实用建议与工具

- **不必死记硬背**：正则表达式语法繁多，如同“字典”，重点掌握基础（`^`, `$`, `.`, `*`, `[ ]`, `\`）即可应对多数场景。
- **遇到复杂表达式时**：
  - 通过网络搜索查询具体符号含义。
  - 使用在线正则测试工具（如 **RegExr**、**regex101.com**）进行解析和验证。
    - 这些工具能实时高亮匹配结果，并解释每部分的含义。
- **注意环境差异**：Shell（如 `grep` 默认使用 **Basic Regular Expressions, BRE**）与其他语言（如 Python 的 `re` 模块）语法可能不同，务必查阅对应文档。
