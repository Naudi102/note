## case 语句概述
`case` 是 Shell 中用于多分支条件判断的一种流程控制结构。它适用于对**单个变量的值进行精确匹配**，并根据匹配结果执行对应的操作。

- 功能固定：仅支持“等于”判断，不支持不等于、大于、小于等比较。
- 语法简洁直观，适合处理有限且明确的选项。
- 执行效率高，代码可读性强。

## case 与 if 的对比
| 特性 | `if` 语句 | `case` 语句 |
|------|----------|------------|
| 判断能力 | 支持复杂逻辑（如 `<`, `>`, `!=`, 正则等） | 仅支持精确值匹配（等于） |
| 语法复杂度 | 较复杂，嵌套多时可读性下降 | 简洁清晰，结构固定 |
| 适用场景 | 多条件、范围判断、逻辑组合 | 枚举型、菜单选择、状态码处理 |

> 视频中强调：`case` 功能不如 `if` 强大，但更直观简洁，因此排在 `if` 之后讲解。

## case 基本语法结构
```bash
case 变量名 in
    值1)
        # 执行命令
        ;;
    值2)
        # 执行命令
        ;;
    *)
        # 默认情况（可选）
        ;;
esac
```

关键点：
- **左括号 `(` 不需要写**，直接写 `值)`。
- 每个分支以 `;;` 结束（表示该分支执行完毕，跳出 `case`）。
- `esac` 是 `case` 的反写，用于结束整个结构。
- 值可以加双引号（如 `"1"`），也可以不加（如 `1`），两者均可。

## case 实战示例
目标：根据用户输入的数字，输出不同人物评价。

```bash
#!/bin/bash

read -p "请输入一个数字: " word

case $word in
    1)
        echo "班长真美"
        ;;
    2)
        echo "蔡老师真美"
        ;;
    *)
        # 默认情况：不输出任何内容
        ;;
esac
```

执行效果：
- 输入 `1` → 输出 “班长真美”
- 输入 `2` → 输出 “蔡老师真美”
- 输入其他值 → 无输出

## 使用 |（或）匹配多个值
`case` 支持用 `|`（竖线）将多个值合并到同一分支，实现“或”逻辑。

示例：
```bash
case $word in
    1|3)
        echo "班长真美"
        ;;
    2|4)
        echo "蔡老师真美"
        ;;
    *)
        ;;
esac
```

说明：
- 输入 `1` 或 `3` → 输出 “班长真美”
- 输入 `2` 或 `4` → 输出 “蔡老师真美”
- `|` 前后**不能有空格**（Shell 语法要求）

> 注意：虽然 `|` 前后的空格非必需，但为了可读性，建议保持紧凑写法（如 `1|3`）。

## 默认情况处理（*）
- 使用 `*` 表示“所有未匹配的情况”，即默认分支。
- `*` **不需要加引号**，直接写 `*)` 即可。
- 若无需默认处理，可省略 `*)` 分支。

## 脚本编写与执行流程
1. 创建脚本文件（如 `case2.sh`）：
   ```bash
   vim case2.sh
   ```
2. 编写 `case` 逻辑（注意缩进和语法）。
3. 保存并退出 Vim。
4. 添加执行权限：
   ```bash
   chmod +x case2.sh
   ```
5. 运行脚本：
   ```bash
   ./case2.sh
   ```
6. 输入测试值，验证输出是否符合预期。

> 视频提示：在 Vim 中粘贴代码后，可利用其格式化功能确保脚本在 Ubuntu 系统中正常运行，避免因换行符或空格问题导致报错。

## AI 总结
`case` 语句是 Shell 脚本中处理多分支选择的高效工具，特别适用于对单一变量进行精确值匹配的场景。虽然其功能不如 `if` 语句强大（不支持范围或逻辑运算），但语法简洁、结构清晰，非常适合实现菜单系统、状态机或命令分发等任务。通过 `|` 可合并多个匹配值，`*` 可处理默认情况，配合良好的脚本编写习惯，能显著提升代码可维护性。
