## Vim 的三种模式概述

Vim 编辑器具有三种核心操作模式：
1. **一般模式（Normal Mode）**：默认启动模式，用于移动光标、复制、删除等操作。
2. **编辑模式（Insert Mode）**：用于实际文本输入和修改。
3. **指令模式（Command Mode）**：通过 `:` 或 `/` 进入，用于执行保存、退出、搜索等命令。

## 编辑模式（Insert Mode）

### 进入编辑模式的常用按键

以下小写字母按键可从一般模式进入编辑模式，适用于不同插入位置：

- **`i`**：在**当前光标位置前**插入文本（Insert）。
- **`a`**：在**当前光标位置后**插入文本（Append），更符合日常编辑习惯。
- **`o`**：在**当前行下方**新开一行并进入编辑模式。

> 建议初学者优先掌握 `i` 和 `a`，因其使用频率最高。

### 大小写按键的区别

大写按键用于特殊位置插入，但使用较少：

- **`I`**（大写 i）：跳到**当前行开头**（忽略缩进空格）开始插入。
- **`A`**（大写 a）：跳到**当前行末尾**开始插入。
- **`O`**（大写 o）：在**当前行上方**新开一行并进入编辑。

> 示例：光标在单词中间时，按 `a` 会向后移一位再插入；按 `i` 则直接在当前位置插入。

## 指令模式（Command Mode）

通过按 `:`（Shift + ;）或 `/` 进入，位于屏幕左下角。

### 保存与退出命令

| 命令 | 作用 |
|------|------|
| `:w` | 保存文件（Write） |
| `:q` | 退出 Vim（Quit） |
| `:wq` | 保存并退出 |
| `:q!` | 强制退出（不保存修改） |

> 所有命令必须**小写**，输入后按 Enter 执行。

### 强制写入与权限问题

- 若文件为**只读**（如 `/etc/passwd`），普通用户无法直接写入。
- 即使使用 `:wq!` 强制写入，仍需具备**写权限**。
- 解决方案：使用 `sudo` 以 root 权限启动 Vim：
  ```bash
  sudo vim /etc/passwd
  ```
  此时可正常保存修改。

> ⚠️ 注意：系统关键文件设为只读是有原因的，非必要勿修改。

### 搜索功能

- **`/关键词`**：向下搜索关键词（如 `/root`）。
- **`n`**：跳转到**下一个**匹配项。
- **`N`**：跳转到**上一个**匹配项。
- **`:noh`**（或 `:nohlsearch`）：关闭搜索结果高亮显示。

### 其他实用指令

| 命令 | 作用 |
|------|------|
| `:set nu` | 显示行号 |
| `:set nonu` | 隐藏行号 |

## 模式切换总结

| 操作 | 命令 |
|------|------|
| **一般模式 → 编辑模式** | 按 `i`、`a`、`o` 等 |
| **编辑模式 → 一般模式** | 按 `Esc` |
| **一般模式 → 指令模式** | 按 `:` 或 `/` |
| **指令模式 → 一般模式** | 执行命令后自动返回，或按 `Esc` 取消 |

> Vim 的核心在于熟练切换三种模式，实现高效文本操作。

本节详细讲解了 Vim 编辑器的**编辑模式**与**指令模式**。编辑模式通过 `i`、`a`、`o` 等键进入，用于文本输入；指令模式通过 `:` 或 `/` 进入，支持保存（`:w`）、退出（`:q`）、搜索（`/`）等高级操作。特别强调了**权限管理**：修改只读系统文件需使用 `sudo`，且应谨慎操作。Vim 虽古老，但在服务器环境和底层编辑中不可替代，掌握其三种模式的切换是 Linux 用户的必备技能。
