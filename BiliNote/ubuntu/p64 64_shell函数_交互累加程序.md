## 交互式累加程序实现

本节教程演示如何在 **Ubuntu** 系统中使用 **Shell 脚本** 编写一个交互式累加程序。该程序允许用户输入一个正整数 $N$，脚本将计算从 1 到 $N$ 的累加和（即 $1 + 2 + \dots + N$），并输出结果。进一步地，通过引入死循环，实现持续交互，无需反复执行脚本。

## 基础函数版本：单次输入累加

### 核心逻辑
- 定义一个名为 `sum` 的函数，用于计算从 1 到输入参数的累加和。
- 使用 `for` 循环遍历从 1 到 $N$ 的所有整数。
- 在循环中不断累加到变量 `s`。
- 最终输出累加结果。

### 脚本结构示例
```bash
#!/bin/bash

sum() {
    local n=$1
    local s=0
    for ((i=1; i<=n; i++)); do
        s=$((s + i))
    done
    echo "累加最终的值为: $s"
}

read -p "请输入你的数字: " n
sum $n
```

### 运行效果
- 用户输入 `100`，程序输出 `5050`（即 $1+2+\dots+100 = 5050$）。

## 增强版：交互式死循环累加

### 改进目标
- 避免每次计算后需重新运行脚本。
- 实现“输入 → 输出 → 再输入”的连续交互体验。

### 实现方式
- 使用 `while true`（或 `while [ 1 -eq 1 ]`）构建死循环。
- 将整个交互逻辑（包括 `read` 和 `sum` 调用）放入循环体内。
- 用户可通过 **Ctrl+C** 强制退出程序。

### 脚本结构示例
```bash
#!/bin/bash

sum() {
    local n=$1
    local s=0
    for ((i=1; i<=n; i++)); do
        s=$((s + i))
    done
    echo "累加最终的值为: $s"
}

while true; do
    read -p "请输入你的数字: " n
    sum $n
done
```

### 运行演示
- 用户依次输入 `100`、`200`、`300`，程序分别返回对应累加结果。
- 输入完成后程序自动回到提示状态，等待下一次输入。

## 注意事项：避免空转消耗资源

- **关键提醒**：`while true` 循环若内部无阻塞操作（如 `read`、`sleep`），会导致 CPU **空转（busy-waiting）**，大量占用系统资源，长期运行可能损害硬件。
- **安全使用条件**：只有当循环体中包含 **等待用户输入**（如 `read`）或 **延时操作**（如 `sleep`）时，才可安全使用死循环。
- 本例中因使用了 `read`，程序会在等待用户输入时挂起，不会持续占用 CPU，因此是安全的。
