## AWK 简介
- **AWK** 是一个功能强大的文本分析工具，能够独立完成复杂的文本处理任务，无需依赖其他命令组合。
- 相比 `cut`、`grep` 等基础命令，AWK 支持：
  - 自定义字段分隔符
  - 行匹配（模式匹配）
  - 字段提取与格式化输出
  - 变量操作与算术运算
  - 在输出前后插入自定义内容

## 基本语法结构
```bash
awk [选项] '模式 {动作}' 文件名
```

常用选项：
- `-F`：指定输入字段分隔符（如 `-F:` 或 `-F'#'`）
- `-v`：定义变量（如 `-v i=1`）

核心组成部分：
- **模式（Pattern）**：决定对哪些行执行操作（如 `/^root/` 匹配以 root 开头的行）
- **动作（Action）**：用 `{}` 包裹，指定具体操作（如 `{print $1, $7}`）
- 特殊关键字：
  - `BEGIN`：在处理第一行前执行（用于输出标题等）
  - `END`：在处理完最后一行后执行（用于输出汇总信息）

字段引用：
- `$1`, `$2`, ..., `$n` 表示第1、2、…、n个字段
- `$0` 表示整行内容

## 实战示例

### 示例1：提取以 root 开头行的第7列
**需求**：从 `/etc/passwd` 文件中，找出以 `root` 开头的行，并输出其第7列（shell路径）。

传统方法需组合多个命令：
```bash
grep "^root" /etc/passwd | cut -d: -f7
```

使用 AWK 更简洁高效：
```bash
awk -F: '/^root/ {print $7}' /etc/passwd
```

> ✅ 优势：单条命令完成匹配+提取，无需管道。

---

### 示例2：输出第1列和第7列，中间用逗号分隔
**需求**：输出所有以 `root` 开头行的用户名（第1列）和 shell（第7列），中间用逗号 `,` 分隔。

AWK 命令：
```bash
awk -F: '/^root/ {print $1 "," $7}' /etc/passwd
```
*![](http://localhost:8000/static/screenshots/screenshot_15ebf152e-763e-4399-83cf-58bc03f0f257.jpg)*

> 💡 技巧：在 `print` 中使用双引号包裹分隔符（如 `","`）可实现自定义格式。

---

### 示例3：在首尾添加自定义文本
**需求**：
- 在输出开头添加一行：`User,Shell`
- 在输出结尾添加一行：`大海哥,病最帅`

AWK 命令：
```bash
awk -F: '
BEGIN {print "User,Shell"}
{print $1 "," $7}
END {print "大海哥,病最帅"}
' /etc/passwd
```
*![](http://localhost:8000/static/screenshots/screenshot_2f967811c-b88b-4cef-aa45-301afd155858.jpg)*

> 🔑 关键点：
> - `BEGIN` 块在读取文件前执行
> - `END` 块在文件处理完毕后执行
> - 所有逻辑必须写在同一对单引号内（或使用多行写法）

---

### 示例4：对用户ID（第3列）加1
**需求**：将 `/etc/passwd` 中每行的用户ID（第3列）数值加1后输出。

AWK 命令：
```bash
awk -F: -v i=1 '{print $1, $3 + i, $7}' /etc/passwd
```

或更直接地：
```bash
awk -F: '{print $1, $3 + 1, $7}' /etc/passwd
```

> 📌 说明：
> - 使用 `-v i=1` 预定义变量 `i`
> - AWK 支持算术运算（如 `$3 + 1`）
> - 输出结果中用户ID均增加1（如原为107 → 显示108）

## AI 总结
AWK 是 Linux 下强大的文本处理工具，集模式匹配、字段提取、变量运算和格式化输出于一体。相比传统命令组合（如 grep + cut），AWK 能以更简洁、高效的单条命令完成复杂任务。本节通过四个典型示例展示了其核心能力：条件筛选、多字段输出、首尾内容注入以及数值计算。掌握 AWK 的基础语法（特别是 `-F`、`BEGIN`/`END`、字段引用和变量操作）可显著提升 Shell 脚本的数据处理效率。
