## 自定义函数的重要性

- Shell 脚本若不能自定义函数，只能重复调用基础命令，程序结构简单且冗余。
- 自定义函数可将通用逻辑封装，提高代码复用性。例如：
  - 累加 1 到 100、1 到 200、1 到 300 等任务，无需为每个需求编写独立脚本。
  - 通过传参方式，一个函数即可处理多种输入场景。

## 函数的基本语法

Shell 函数定义有两种形式，**推荐显式使用 `function` 关键字**以增强可读性：

```bash
# 推荐写法（带 function 关键字）
function func_name() {
    # 函数体
}

# 可选写法（不带 function）
func_name() {
    # 函数体
}
```

- 函数名后必须跟一对小括号 `()`。
- 函数体由大括号 `{}` 包裹。
- 函数可以为空（即无任何语句）。

## 函数的声明与调用顺序

- **Shell 脚本是逐行解释执行的**，不支持像 C/Java 那样的“先声明后定义”或编译时符号解析。
- **必须先声明函数，再调用**；否则会报“command not found”错误。

## 函数的返回值机制

- Shell 函数的返回值通过 `return` 语句指定，但**仅支持 0–255 的整数**：
  - `0` 表示成功；
  - 非零值（1–255）表示不同类型的错误。
- **若未显式使用 `return`**，则默认返回**函数中最后一条命令的退出状态码**。
- **重要限制**：`return` **不能用于返回计算结果（如数值、字符串）**，仅用于状态码。

> ✅ 正确做法：使用 `echo` 输出结果，调用时通过命令替换 `$()` 或反引号 `` ` `` 捕获。

## 函数参数与输出方式

- 函数参数通过位置变量 `$1`, `$2`, ..., `$n` 访问，与脚本参数用法一致。
- **返回计算结果应使用 `echo`**，而非 `return`。
  ```bash
  function add() {
      sum=$(( $1 + $2 ))
      echo $sum
  }
  ```
- 调用时捕获输出：
  ```bash
  result=$(add 10 20)
  echo $result
  ```

## 完整示例：实现两数相加的函数

### 脚本内容 (`3.sh`)
```bash
#!/bin/bash

function add() {
    sum=$(( $1 + $2 ))
    echo $sum
}

read -p "请输入第一个数: " n1
read -p "请输入第二个数: " n2

result=$(add $n1 $n2)
echo "等于 $result"
```

### 执行步骤
1. 创建脚本文件：`vim 3.sh`
2. 添加上述内容，并确保首行有 `#!/bin/bash`
3. 赋予执行权限：`chmod +x 3.sh`
4. 运行脚本：`./3.sh`

### 示例运行
```
请输入第一个数: 100
请输入第二个数: 200
等于 300
```

> 💡 注意：若忘记添加 `#!/bin/bash`，可能导致在某些环境中无法正确解析。
