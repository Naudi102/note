## 系统变量概述

在 Shell 脚本编程中，除了自定义变量外，系统本身也预定义了许多**系统变量（System Variables）**。这些变量由操作系统或 Shell 环境预先设置，用于存储关键信息（如当前用户、工作目录、路径等），便于脚本调用和系统交互。

> 系统变量是“前人已经造好的轮子”，开发者应优先了解并利用，避免重复造轮子。

## 系统变量的命名规范

- **系统变量通常全部使用大写字母命名**，例如：
  - `HOME`
  - `PWD`
  - `PATH`
- 这种命名约定有助于快速识别哪些是系统预定义变量。

## 如何引用系统变量

在 Shell 脚本或终端中，通过 `$` 符号加变量名来引用系统变量：

```bash
echo $HOME
echo $PWD
echo $PATH
```

- `$HOME`：显示当前用户的主目录路径。
- `$PWD`：显示当前工作目录（Print Working Directory）。
- `$PATH`：显示系统查找可执行文件的目录列表。

## 查看所有系统变量的方法

要查看当前 Shell 环境中**所有已定义的变量（包括系统变量和环境变量）**，可使用 `set` 命令：

```bash
set
```

- 该命令输出所有变量及其值。
- 变量名在前（通常为大写），等号 `=` 后为对应的值。
- 某些变量值较长，可能跨多行显示。

> 注意：`set` 是小写命令，所有 Shell 命令均为小写。

![](http://localhost:8000/static/screenshots/screenshot_2cb409950-835a-4700-8c92-2c049166462b.jpg)*

## 避免变量名冲突的建议

为防止自定义变量与系统变量冲突，建议：

- **自定义变量使用小写字母命名**（如 `my_var`、`username`）。
- **避免全大写命名自定义变量**（如 `MY_VAR` 可能与未来系统变量冲突）。
- 此做法可有效减少命名冲突风险，提升脚本健壮性与可维护性。

> 示例：若你叫“张三”，变量名应写作 `zhangsan` 而非 `ZHANGSAN`。
